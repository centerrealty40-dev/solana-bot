## Статус Phase 0 (реализовано)

| Элемент | Где |
|--------|-----|
| Entry | `src/scripts/live-oscar.ts` → `npm run live-oscar` |
| Конфиг | `src/live/config.ts` → `loadLiveOscarConfig()` |
| Журнал | `src/live/store-jsonl.ts` |
| Цикл | `src/live/main.ts` → heartbeat, SIGINT/SIGTERM |
| PM2 | `ecosystem.config.cjs` → приложение **`live-oscar`** (по умолчанию выкл.) |
| Тест guard путей | `tests/live-oscar-config.test.ts` |

**Запуск локально:** `LIVE_STRATEGY_ENABLED=0 LIVE_TRADES_PATH=./data/live/test-live.jsonl npm run live-oscar`  
**На VPS:** только `pm2 start ecosystem.config.cjs --only live-oscar` когда будете готовы; до включения торговли оставляйте `LIVE_STRATEGY_ENABLED=0`.

**Открытые решения §12 спеки** (до Phase 3–4): общий ли Jupiter Pro URL/key; обязательность reconcile при старте; учёт `simulate` в qn-feature meter.
