Создать **3 миграции** + Drizzle-схему для таблиц, которые потребуются последующим этапам W6 (DEX-коллекторам, jupiter-watcher, direct-LP-detector). **Точное число таблиц ПОСЛЕ:**
- **До:** 12 таблиц.
- **+4** `*_pair_snapshots` (raydium, meteora, orca, moonshot)
- **+1** `jupiter_route_snapshots`

## Если что-то пойдёт не так

- **`db:migrate` падает на VPS:** не пытаться руками править таблицы; вернуться, спросить оркестратора. Хотфикс — отдельный коммит.
- **`db:generate` упорно создаёт лишний SQL:** см. шаг 6 — это нормально для drizzle-kit, нужно удалить лишнее и переименовать snapshot. Если после двух попыток cleanup не работает — ESCALATE.
- **PM2-процесс упал после миграции:** проверить логи (`pm2 logs <name> --nostream --lines 50`), но НЕ пытаться чинить — это вне scope spec. ESCALATE.
- **Не работает SSH к VPS:** руководствоваться правилом `server-autodeploy.mdc` (рассказать пользователю, не делать manual fallback).
