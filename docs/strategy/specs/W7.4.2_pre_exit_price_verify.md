# Spec W7.4.2 — Pre-exit price verification (Jupiter token→SOL)

**Назначение:** перед частичной продажей (TP ladder) и полным закрытием сравнить **исполнимую** цену выхода (Jupiter quote **mint → WSOL**) с **ценой снапшота трекера** (`curMetric`), по тем же порогам slip/impact, что W7.4 pre-entry (`PAPER_PRICE_VERIFY_MAX_*`, `TIMEOUT`).

**Реализация (код):**

- `src/papertrader/pricing/price-verify.ts` — `jupiterQuoteSellPriceUsd`, `verifyExitPrice`.
- `src/papertrader/executor/tracker.ts` — вызов до `livePhase4.tryTokenToSolSell` и до учёта paper `partial_sell` / `close`.
- Конфиг: `priceVerifyExitEnabled`, `priceVerifyExitBlockOnFail` (`PAPER_PRICE_VERIFY_EXIT_*`).

**Не в scope (явно):**

- **LIQ_DRAIN** / **NO_DATA** — pre-exit не вызывается (аварийный или отсутствие цены).
- Отдельные пороги для exit — v1 переиспользует лимиты entry.

**JSONL:**

- `eval-skip-exit` — отложенный выход при `blocked` и `PAPER_PRICE_VERIFY_EXIT_BLOCK_ON_FAIL=1`.
- Поле `priceVerifyExit` на `partial_sell` и `close`, когда verify включён (включая `skipped` по сети).
- `heartbeat.skippedPriceVerifyExit` — счётчик отложенных выходов за сессию процесса.

**Откат:** `PAPER_PRICE_VERIFY_EXIT_ENABLED=0` или `EXIT_BLOCK_ON_FAIL=0` (shadow: стамп без блокировки при сохранении verify enabled — только если не ставить block; при verify off полей нет).
