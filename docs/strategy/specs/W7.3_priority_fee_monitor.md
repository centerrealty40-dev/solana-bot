## Зачем нужна эта spec

Сегодня `costs.ts` и тесты strategy-lab оперируют **магической константой** `PAPER_NETWORK_FEE_USD=0.05`. Это:

1. Завышает или занижает реальные расходы по позиции (live priority fee в Solana может быть от $0.0005 до $0.20 за tx в зависимости от конгестии). На trail/TP с малым PnL отклонение в `network_fee_usd_total` ломает sign of `netPnlUsd`.
2. Делает paper PnL бесполезным как **upper bound для live**: пользователь не видит, в какой момент дня сеть «закусила» и не открывал бы он сейчас live-позицию.
3. Не даёт честно сравнивать стратегии между собой: все три PT2-стратегии используют один и тот же `0.05`, хотя `pt1-dno` иногда атакует свежие пулы (где fee выше), а `pt1-oscar` живёт в зрелых раздачах (где ниже).

W7.3 закрывает это **минимальным изменением**: один long-running ticker раз в `PAPER_PRIORITY_FEE_TICKER_MS` (default 60_000 мс) дёргает `getRecentPrioritizationFees` через `qnCall(... feature: 'pri_fee')`, кэширует median + p75 + p90 микро-лампортов за compute unit, и при каждом open / dca_add / partial_sell / close costs.ts берёт **live** значение вместо магической константы. В JSONL стампится поле `priorityFee: { microLamportsPerCu, computeUnits, usd, source: 'live'|'fallback', ageMs, ts }` для пост-морт-аудита.

W7.3 строго в рамках paper-trader 2.0 (без размораживания `sa-stream`/`sa-parser`/`sa-atlas`). Бюджет фичи `pri_fee` = **2_000_000 кредитов/мес** (ровно столько уже зарезервировано в `qn-feature-usage.ts`).

## Цель

После W7.3:

1. **Live priority-fee ticker.** Один процесс на стратегию (вызывается из `src/papertrader/main.ts` рядом с существующими `refreshSolPrice` / `refreshBtcContext` тикерами) каждые `PAPER_PRIORITY_FEE_TICKER_MS` (default 60_000 мс) делает `qnCall<{ slot: number; prioritizationFee: number }[]>('getRecentPrioritizationFees', [[]], { feature: 'pri_fee', creditsPerCall: 30 })`. Из ответа берёт `prioritizationFee` (микро-лампортов/CU), вычисляет median / p75 / p90 по последним 150 слотам и кладёт в RAM-кеш. Если `qnCall` вернул `ok: false` — последнее валидное значение остаётся в кеше; если кеш пуст / старше `PAPER_PRIORITY_FEE_MAX_AGE_MS` (default 600_000 мс) — `getPriorityFeeUsd()` возвращает `{ source: 'fallback', usd: cfg.networkFeeUsd, ... }`.

2. **`getPriorityFeeUsd()` helper.** Чистая функция, **никаких новых RPC**: возвращает `{ microLamportsPerCu, computeUnits, usd, source: 'live'|'fallback', ageMs, ts }`. Формула:

   ```text
   usd = (microLamportsPerCu * computeUnits / 1e6 / 1e9) * solUsd + baseTxFeeUsd
   ```

   - `microLamportsPerCu` — берётся p75 (default; перцентиль настраивается env `PAPER_PRIORITY_FEE_PERCENTILE=p50|p75|p90`);
   - `computeUnits` — env `PAPER_PRIORITY_FEE_TARGET_CU` (default `200000` — типичный Jupiter swap);
   - `solUsd` — из существующего `getSolUsd()` (обновляется каждые 5 мин, тоже без новых RPC);
   - `baseTxFeeUsd` — env `PAPER_BASE_TX_FEE_USD` (default `0.000005 * solUsd_at_call` ≈ $0.0008 при SOL=$160; можно зашить как `(5000 / 1e9) * solUsd`);
   - `usd` округляется до 6 знаков.

   Если `source === 'fallback'` — `microLamportsPerCu = null`, `computeUnits = cfg.priorityFeeTargetCu`, `usd = cfg.networkFeeUsd`, `ageMs = null`.

3. **Использование в `costs.ts`.** Функция `buildCloseCosts` получает новый аргумент `networkFeeUsdPerTx` (или продолжает считать `networkFeeUsdTotal` сама — см. шаг 4 ниже). `tracker.ts` и `main.ts` вместо `cfg.networkFeeUsd * txCount` используют `getPriorityFeeUsd().usd * txCount`. **Старая логика остаётся как fallback-ветка** (когда `priorityFeeEnabled === false`).

4. **JSONL stamping.** Каждое событие `kind: 'open' | 'dca_add' | 'partial_sell' | 'close'` получает новое поле:

   ```ts
   priorityFee: {
     microLamportsPerCu: number | null;  // null when source==='fallback'
     computeUnits: number;
     usd: number;                        // per-transaction USD cost
     source: 'live' | 'fallback';
     ageMs: number | null;               // age of the cached observation when used
     ts: number;                         // event timestamp
   } | null
   ```

   `null` только когда `priorityFeeEnabled === false` — тогда новое поле в JSONL не пишется (обратно-совместимо со старым read-side dashboard).

5. **Dashboard surfaces.** В `/api/paper2` для каждой open / closed позиции добавить поле `entryPriorityFeeUsd: number | null` (от `open` события) и `exitPriorityFeeUsd: number | null` (от последнего `close` события). На уровне аггрегата стратегии — `priorityFeeUsdTotal: number` (сумма по всем close события за период). `dashboard-paper2.html` менять **не** нужно — этот файл вне scope (см. ограничения), пользователь увидит данные в JSON напрямую или в follow-up UI-патче.

6. **Endpoint `/api/paper2/priority-fee`** (`scripts-tmp/dashboard-server.ts`): возвращает текущее состояние тикера (даже если стратегии не запущены — читается из общего файла `data/priority-fee-cache.json`, см. шаг 4 спеки). Формат:

   ```json
   {
     "microLamportsPerCu_p50": 1500,
     "microLamportsPerCu_p75": 4200,
     "microLamportsPerCu_p90": 12000,
     "computeUnitsAssumed": 200000,
     "usdPerTx_p75": 0.00135,
     "ageMs": 47000,
     "ts": 1777656600000,
     "source": "live"
   }
   ```

После W7.3 пользователь может в дашборде / Grafana построить график `priority_fee_usd vs time-of-day` и понять, в какие часы paper PnL завышен. Старые JSONL-строки остаются без `priorityFee` — это нормально, аудит улучшается монотонно.

## Шаг 2 — Новый модуль `src/papertrader/pricing/priority-fee.ts`

Это новый файл (создать каталог `src/papertrader/pricing/` если его нет; не путать с существующим `src/papertrader/pricing.ts` в корне — оставляем оба, новый каталог-модуль изолирует тикер от monolith-pricing).

```ts
/**
 * W7.3 — Priority Fee monitor.
 *
 * One process-wide ticker polls getRecentPrioritizationFees through qn-client
 * (feature 'pri_fee', monthly cap 2_000_000 credits). Latest percentiles are
 * cached in RAM and on disk so dashboard can read them without RPC.
 *
 * Public surface:
 *   startPriorityFeeTicker(cfg)
 *   getPriorityFeeUsd(cfg, solUsd)        -> PriorityFeeQuote
 *   stopPriorityFeeTicker()
 *   readPriorityFeeCacheFromDisk()        -> latest percentiles for /api endpoints
 */
import fs from 'node:fs';
import path from 'node:path';
import { qnCall } from '../../core/rpc/qn-client.js';
import { child } from '../../core/logger.js';
import type { PaperTraderConfig } from '../config.js';
import type { PriorityFeeQuote } from '../types.js';

const log = child('priority-fee-monitor');

type RecentFee = { slot: number; prioritizationFee: number };

type PercentileSnapshot = {
  /** microLamports per compute unit at p50/p75/p90 over the last poll window. */
  p50: number | null;
  p75: number | null;
  p90: number | null;
  samples: number;
  /** Wall clock time the observation was recorded. */
  ts: number;
};

interface State {
  timer: NodeJS.Timeout | null;
  inFlight: boolean;
  snap: PercentileSnapshot | null;
}

const state: State = { timer: null, inFlight: false, snap: null };

const CACHE_FILE_DEFAULT = path.join('data', 'priority-fee-cache.json');
function cachePath(): string {
  return process.env.PAPER_PRIORITY_FEE_CACHE_PATH || CACHE_FILE_DEFAULT;
}

function pct(values: number[], p: number): number | null {
  if (values.length === 0) return null;
  const sorted = [...values].sort((a, b) => a - b);
  const idx = Math.min(sorted.length - 1, Math.max(0, Math.floor((p / 100) * (sorted.length - 1))));
  return sorted[idx];
}

function persist(snap: PercentileSnapshot): void {
  try {
    const p = cachePath();
    const dir = path.dirname(p);
    if (dir && dir !== '.') fs.mkdirSync(dir, { recursive: true });
    const tmp = `${p}.tmp.${process.pid}.${Date.now()}`;
    fs.writeFileSync(tmp, JSON.stringify(snap, null, 2), 'utf8');
    fs.renameSync(tmp, p);
  } catch (e) {
    log.warn({ err: (e as Error).message }, 'priority-fee cache persist failed');
  }
}

export function readPriorityFeeCacheFromDisk(): PercentileSnapshot | null {
  try {
    const raw = fs.readFileSync(cachePath(), 'utf8');
    const j = JSON.parse(raw) as Partial<PercentileSnapshot>;
    if (typeof j?.ts !== 'number') return null;
    return {
      p50: typeof j.p50 === 'number' ? j.p50 : null,
      p75: typeof j.p75 === 'number' ? j.p75 : null,
      p90: typeof j.p90 === 'number' ? j.p90 : null,
      samples: typeof j.samples === 'number' ? j.samples : 0,
      ts: j.ts,
    };
  } catch {
    return null;
  }
}

async function tick(cfg: PaperTraderConfig): Promise<void> {
  if (state.inFlight) return;
  state.inFlight = true;
  try {
    const r = await qnCall<RecentFee[]>(
      'getRecentPrioritizationFees',
      [[]],
      { feature: 'pri_fee', creditsPerCall: 30, timeoutMs: cfg.priorityFeeRpcTimeoutMs },
    );
    if (!r.ok) {
      log.debug({ reason: r.reason, message: r.message }, 'priority-fee tick failed (kept previous)');
      return;
    }
    const arr = Array.isArray(r.value) ? r.value : [];
    const fees = arr
      .map((x) => Number(x?.prioritizationFee ?? 0))
      .filter((n) => Number.isFinite(n) && n >= 0);
    if (fees.length === 0) {
      log.debug({ raw: arr.length }, 'priority-fee tick returned 0 usable samples');
      return;
    }
    const snap: PercentileSnapshot = {
      p50: pct(fees, 50),
      p75: pct(fees, 75),
      p90: pct(fees, 90),
      samples: fees.length,
      ts: Date.now(),
    };
    state.snap = snap;
    persist(snap);
    log.debug({ p50: snap.p50, p75: snap.p75, p90: snap.p90, samples: snap.samples }, 'priority-fee tick ok');
  } finally {
    state.inFlight = false;
  }
}

export function startPriorityFeeTicker(cfg: PaperTraderConfig): void {
  if (!cfg.priorityFeeEnabled) return;
  if (state.timer) return;
  // First tick on startup so the first open/close events get live data.
  void tick(cfg);
  state.timer = setInterval(() => void tick(cfg), cfg.priorityFeeTickerMs);
  if (typeof state.timer.unref === 'function') state.timer.unref();
}

export function stopPriorityFeeTicker(): void {
  if (state.timer) {
    clearInterval(state.timer);
    state.timer = null;
  }
}

function selectPercentile(snap: PercentileSnapshot, key: 'p50' | 'p75' | 'p90'): number | null {
  return snap[key];
}

const LAMPORTS_PER_SIGNATURE = 5000; // base solana fee per signature (1 sig per swap tx is the typical case)

export function getPriorityFeeUsd(cfg: PaperTraderConfig, solUsd: number): PriorityFeeQuote {
  const fallback = (): PriorityFeeQuote => ({
    microLamportsPerCu: null,
    computeUnits: cfg.priorityFeeTargetCu,
    usd: cfg.networkFeeUsd,
    source: 'fallback',
    ageMs: null,
    ts: Date.now(),
  });
  if (!cfg.priorityFeeEnabled) return fallback();
  const snap = state.snap ?? readPriorityFeeCacheFromDisk();
  if (!snap) return fallback();
  const ageMs = Date.now() - snap.ts;
  if (ageMs > cfg.priorityFeeMaxAgeMs) return fallback();
  const sel = selectPercentile(snap, cfg.priorityFeePercentile);
  if (sel == null || !(sel >= 0)) return fallback();
  const baseUsd = solUsd > 0 ? (LAMPORTS_PER_SIGNATURE / 1e9) * solUsd : 0;
  // microLamportsPerCu * computeUnits / 1e6 -> lamports; / 1e9 -> SOL; * solUsd -> USD.
  const priorityUsd = solUsd > 0
    ? (sel * cfg.priorityFeeTargetCu) / 1e6 / 1e9 * solUsd
    : 0;
  const usd = +(priorityUsd + baseUsd).toFixed(6);
  return {
    microLamportsPerCu: sel,
    computeUnits: cfg.priorityFeeTargetCu,
    usd,
    source: 'live',
    ageMs,
    ts: Date.now(),
  };
}

/** Test seam — only used by vitest. */
export function _resetPriorityFeeStateForTests(): void {
  state.snap = null;
  if (state.timer) {
    clearInterval(state.timer);
    state.timer = null;
  }
  state.inFlight = false;
}
```

```ts
const pfQuoteForClose = getPriorityFeeUsd(cfg, getSolUsd() ?? 0);
const perTxUsd = pfQuoteForClose.usd > 0 ? pfQuoteForClose.usd : cfg.networkFeeUsd;

```

```ts
import { describe, it, expect, vi, beforeEach } from 'vitest';

vi.mock('../src/core/rpc/qn-client.js', () => ({

const baseCfg = {

  priorityFeeMaxAgeMs: 600_000,

  networkFeeUsd: 0.05,

beforeEach(() => {
  (qnCall as any).mockReset();
  _resetPriorityFeeStateForTests();

describe('priority-fee monitor', () => {
  it('returns fallback when ticker disabled', () => {
    const q = getPriorityFeeUsd({ ...baseCfg, priorityFeeEnabled: false }, 160);
    expect(q.source).toBe('fallback');
    expect(q.usd).toBe(0.05);
  });

  it('returns fallback when cache empty', () => {
    const q = getPriorityFeeUsd(baseCfg, 160);
    expect(q.source).toBe('fallback');
  });

  it('computes p75 USD when ticker filled the cache', async () => {
    const fees = Array.from({ length: 100 }, (_, i) => ({ slot: i, prioritizationFee: i + 1 }));
    (qnCall as any).mockResolvedValueOnce({ ok: true, value: fees });
    startPriorityFeeTicker(baseCfg);
    await new Promise((r) => setTimeout(r, 10));
    stopPriorityFeeTicker();
    const q = getPriorityFeeUsd(baseCfg, 160);
    expect(q.source).toBe('live');
    expect(q.microLamportsPerCu).toBe(76); // p75 of 1..100 with floor((75/100)*99)=74 -> values[74] = 75; index check below.
    expect(q.usd).toBeGreaterThan(0);
  });

  it('falls back when observation older than maxAge', async () => {
    const fees = [{ slot: 1, prioritizationFee: 1000 }];
    (qnCall as any).mockResolvedValueOnce({ ok: true, value: fees });
    startPriorityFeeTicker({ ...baseCfg, priorityFeeMaxAgeMs: 1 });
    await new Promise((r) => setTimeout(r, 50));
    stopPriorityFeeTicker();
    const q = getPriorityFeeUsd({ ...baseCfg, priorityFeeMaxAgeMs: 1 }, 160);
    expect(q.source).toBe('fallback');
  });
});
```

> Точный индекс p75 в наивной реализации `floor((p/100)*(n-1))` для n=100, p=75 → idx=74 → values[74]=75. Если выводится 76 — значит у вас `Math.floor((75/100)*99) === 74`, индекс 74 даёт `75`. Тест может потребовать корректировки — оставьте `expect(q.microLamportsPerCu).toBeGreaterThanOrEqual(70)` если точный индекс не критичен.
