## Зачем нужна эта spec

### Главная мотивация (бизнес)

Сегодня paper-trader 2.0 закрывает позицию **только** по сигналам прайсового tracker'а (`TP`, `SL`, `TRAIL`, `TIMEOUT`, `KILLSTOP`, `NO_DATA`). Это значит, что при классическом soft-rug (LP partially withdrawn → quote depth drops 80 % в одном слоте, цена ещё не упала, но любой sell проскользит на 50–90 %) трейдер **не предпринимает никаких действий**, пока сама цена не свалится. К этому моменту реальный live-выход стоит больше, чем номинал позиции.

Реальные наблюдения по `pt1-dno` (manual audit на 200 closed-позициях за 7 дней): **9 % closed-сделок** имели exit `TIMEOUT` или `SL` с `net_pnl_usd ≈ −12..−25 %`, при этом **в момент close** Dexscreener `liq_usd` упал в 5-30 раз от entry — это были тихие rug-pulls. Live на этих позициях получил бы −80..−100 %.

### Что меняет W7.5

После W7.5 каждый `trackerTick()` для каждой open-позиции:

1. Берёт **свежий** `liq_usd` пула (приоритет — DEX snapshot из БД, fallback — `getMultipleAccounts(vault_a, vault_b)` через `qnCall feature='liq_watch'`).
2. Сравнивает с `entryLiqUsd` (стампится в `open` event на основе `features.liq_usd`).
3. Если относительное падение `(entryLiqUsd - currentLiqUsd) / entryLiqUsd > cfg.liqWatchDrainPct` (default 35 %) **и** позиция жива хотя бы `cfg.liqWatchMinAgeMin` (default 1) минут — force-close с `exitReason: 'LIQ_DRAIN'`.
4. Дополнительная защита от false-positive (когда коллектор временно отстал и snapshot слишком старый): требуется **N** последовательных «провалов» подряд (default 2 tick'а = 60 сек устойчиво ниже порога).

W7.5 покрывает:
* классический rug-pull (LP burn after creator sells whole bag);
* soft-rug (creator drains 80 % vault, оставив 20 % ликвидности — цена держится на бумаге, но live exit невозможен);
* slow death (liquidity линейно тает 30 минут — TP/SL не срабатывают, но pool становится нерабочим).

### Operational

* Бюджет фичи `liq_watch` = **12 000 000 кредитов/мес** (зарезервировано в `qn-feature-usage.ts` с W7.1, до сих пор не использовалось).
* План траты: 30 cr × `getMultipleAccounts(2 accounts)` (1 RPC = 30 cr) на 1 фолбэк-тик. При 5 open positions × 1 фолбэк-тик в 5 мин (если snapshot устарел) × 24 × 30 = ~43k cr/мес. С запасом до бюджета.
* Если используем DB snapshots only (default), бюджет вообще не тратится.
* После W7.5 фактический расход QN суммарно: pri_fee ~1.3M + safety ~50k + liq_watch (fallback only) ~50k = ~1.4M из 80M бюджета. Бюджет огромен, фича безопасна.

### Почему именно сейчас (после W7.4)

W7.4 дал нам **realistic entry price**. Без W7.5 paper всё ещё игнорирует тот факт, что **никакая цена входа не имеет смысла, если live exit невозможен из-за пустого пула**. Метрика «paper PnL = upper bound для live» закрывается на 80 % после W7.5.

W7.6 (sim/`simulateTransaction`) будет лучше работать поверх W7.5: zero-liquidity позиции уже отфильтрованы.

## Цель

После W7.5:

1. **Поле `entryLiqUsd: number | null` в `open` event.** Берётся из `features.liq_usd` (которое уже считается дискавери-ловерами) либо из `*_pair_snapshots.liq_usd` (если features его не хранят). Если оба источника пустые — `null` (тогда liq-watch для этой позиции автоматически отключается).

2. **Поле `pairAddress: string | null` в `open` event.** Канонический адрес пула (Raydium/Meteora/Orca/Moonshot). Берётся из `features.pair_address` — оно уже есть в discovery rows (см. `src/papertrader/discovery/dip-clones.ts` → `buildFeatures`).

3. **Новый модуль `src/papertrader/pricing/liq-watch.ts`.** Public surface:
   * `loadCurrentPoolLiqUsd(pairAddress: string, source: DexSource): Promise<{ liqUsd: number | null; ageMs: number; from: 'snapshot'|'rpc'|'none' }>` — главная функция. Сначала пробует pg-snapshot (last `liq_usd` в `*_pair_snapshots` за последние `cfg.liqWatchSnapshotMaxAgeMs`), потом — RPC fallback (если включено `cfg.liqWatchRpcFallback === true`). RPC fallback использует `qnCall<{ value: AccountInfo[] }>('getMultipleAccounts', [[vaultA, vaultB], { encoding: 'jsonParsed' }], { feature: 'liq_watch', creditsPerCall: 30 })` — для этого нужны `vault_a`/`vault_b` адреса; брать из `*_pair_snapshots` (колонка `vault_a_address` / `vault_b_address` для Raydium; для Meteora — другая схема, см. § "Знание адресов вольтов" ниже).
   * `evaluateLiqDrainState(args)` — чистая функция, работает на in-memory state без I/O. Принимает `{ entryLiqUsd, currentLiqUsd, ageMs, consecutiveFailures, cfg }`. Возвращает `LiqWatchVerdict`.

4. **Интеграция в `trackerTick`** (`src/papertrader/executor/tracker.ts`): для каждой open-позиции **перед** прайсовыми проверками (TP/SL/...) дёрнуть `loadCurrentPoolLiqUsd` (если `cfg.liqWatchEnabled && ot.pairAddress && ot.entryLiqUsd > 0`). Если `evaluateLiqDrainState` вернул `{ kind: 'force-close', reason: 'LIQ_DRAIN' }` — закрыть позицию через тот же `buildClosedTrade` pipeline что и обычный close (с `exitReason: 'LIQ_DRAIN'`), стампя в close-event поле `liqWatch: { entryLiqUsd, currentLiqUsd, dropPct, ageMs, source, ts }`. Прайсовые проверки в этом тике пропускаются. Если verdict `{ kind: 'ok' }` или `{ kind: 'pending' }` — продолжать обычный track.

5. **Stamping в JSONL.** Каждый `kind: 'open'` event получает `entryLiqUsd: number | null` и `pairAddress: string | null` (даже если `liqWatchEnabled === false` — для аудита будущих фич). Каждый `kind: 'close'` event с `exitReason: 'LIQ_DRAIN'` (а также любой close, если `cfg.liqWatchStampOnAllClose === true`) получает `liqWatch: { source, entryLiqUsd, currentLiqUsd, dropPct, ageMs, consecutiveFailures, ts }`. Опционально (если `cfg.liqWatchStampOnTrack === true`, default false): каждый трекер-тик пишет отдельный `kind: 'liq_watch_tick'` event с тем же payload — это даёт временной ряд для дебага, но ↑ JSONL volume в 5-10 раз; default off.

6. **Dashboard surfaces.**
   * В `EnrichedOpen`: `entryLiqUsd: number | null`, `currentLiqUsd: number | null`, `liqDropPct: number | null` (live, рассчитывается на лету из последнего тика).
   * В `Paper2ClosedRow`: `exitLiqUsd: number | null`, `exitLiqDropPct: number | null` (только для close с `liqWatch` стампом).
   * Per-strategy aggregate: `liqDrainExitsCount: number` (сколько LIQ_DRAIN exits за окно), `liqDrainAvgDropPct: number | null`.
   * `exitReason: 'LIQ_DRAIN'` появляется в существующем enum для UI.

7. **Dashboard UI** (`scripts-tmp/dashboard-paper2.html`): рядом с `entry slip` (W7.4) добавить `liq drop Xn%` бейдж (зелёный <10 %, жёлтый 10-25 %, красный ≥25 %; n/a если liqWatch off). В блок closed-позиций — иконка `LIQ_DRAIN` для соответствующих exits. В заголовок стратегии — `liq_drain N exits / Yn% avg drop`.

8. **Endpoint `GET /api/paper2/liq-watch-stats`** (`scripts-tmp/dashboard-server.ts`): аналогично W7.4 endpoint, возвращает per-strategy агрегаты за окно:

## Знание адресов вольтов (важная подзадача)

В текущей версии paper-trader не сохраняет `vault_a_address` / `vault_b_address` пула в JSONL — только `pair_address`. Чтобы RPC-fallback мог дёрнуть `getMultipleAccounts`, надо:

* **MVP-вариант (default, рекомендуемый):** RPC fallback **отключён** (`cfg.liqWatchRpcFallback = false`). Используем только snapshots. Если все коллекторы здоровы — этого хватает, snapshots обновляются каждые 30 сек на mint, который попадал в их sweep. Для долгоживущих позиций (>1 ч) есть risk, что mint выпадет из top-list коллектора и snapshot перестанет освежаться → fallback на `paper2-open-snapshot-enrich.mjs`, который **уже** это патчит (см. W7.2).
* **Расширенный вариант (опциональный, env `PAPER_LIQ_WATCH_RPC_FALLBACK=1`):** дёргаем `getMultipleAccounts` по `vault_a_address`/`vault_b_address`. Эти адреса нужно стампить в `open` event как `vaults: { a: string, b: string } | null` — взять из существующих колонок таблицы `raydium_pair_snapshots.vault_a_address` / `vault_b_address` (для Raydium они есть). Для Meteora / Orca / Moonshot — стоит проверить наличие этих колонок; если их нет, RPC fallback просто пропускается (`from: 'none'`) и используется только snapshot.

В рамках W7.5 **MVP** (`PAPER_LIQ_WATCH_RPC_FALLBACK=0`) — если позже понадобится более полное покрытие, отдельная W7.5.1 спека добавит vault-stamping для Meteora/Orca/Moonshot.

## Шаг 2 — Новый модуль `src/papertrader/pricing/liq-watch.ts`

```ts
/**
 * W7.5 — Liquidity Drain Watch.
 *
 * Per-tracker-tick check of pool liquidity_usd vs entry baseline. Default
 * source is the DB pair-snapshot (free, written by sa-{raydium,meteora,
 * orca,moonshot}-collector). Optional RPC fallback via qnCall(feature='liq_watch').
 *
 * Pure functions only — no global timers. State (consecutiveFailures) lives
 * on the OpenTrade object owned by tracker.ts.
 */
import postgres from 'postgres';
import { qnCall } from '../../core/rpc/qn-client.js';
import { child } from '../../core/logger.js';
import type { PaperTraderConfig } from '../config.js';
import type { LiqWatchVerdict, DexSource } from '../types.js';

const log = child('liq-watch');

// One module-scoped sql client. Reuses existing connection pool through env DATABASE_URL.
let sqlClient: ReturnType<typeof postgres> | null = null;
function getSql(): ReturnType<typeof postgres> {
  if (sqlClient) return sqlClient;
  const url = process.env.DATABASE_URL || '';
  sqlClient = postgres(url, { max: 4, idle_timeout: 30, connect_timeout: 5 });
  return sqlClient;
}

const TABLE_BY_SOURCE: Record<DexSource, string | null> = {
  raydium: 'raydium_pair_snapshots',
  meteora: 'meteora_pair_snapshots',
  orca: 'orca_pair_snapshots',
  moonshot: 'moonshot_pair_snapshots',
  pump: null,
  jupiter: null,
};

export interface LoadLiqArgs {
  pairAddress: string;
  source: DexSource;
  cfg: PaperTraderConfig;
}

export interface LoadLiqResult {
  liqUsd: number | null;
  ageMs: number;
  from: 'snapshot' | 'rpc' | 'none';
}

export async function loadCurrentPoolLiqUsd(args: LoadLiqArgs): Promise<LoadLiqResult> {
  const { pairAddress, source, cfg } = args;
  const ts = Date.now();
  const table = TABLE_BY_SOURCE[source];
  if (!table) return { liqUsd: null, ageMs: 0, from: 'none' };
  try {
    const sql = getSql();
    const rows = await sql<{ liq_usd: string | number | null; bucket_ts: Date }[]>`
      select liq_usd, bucket_ts
      from ${sql(table)}
      where pair_address = ${pairAddress}
      order by bucket_ts desc
      limit 1
    `;
    if (rows.length === 0) {
      // No snapshot row — try RPC fallback if enabled.
      return await maybeRpcFallback(args, ts);
    }
    const row = rows[0];
    const liqUsd = row.liq_usd != null ? Number(row.liq_usd) : null;
    const ageMs = Math.max(0, ts - new Date(row.bucket_ts).getTime());
    if (ageMs > cfg.liqWatchSnapshotMaxAgeMs) {
      // Snapshot too old — try RPC fallback.
      const rpc = await maybeRpcFallback(args, ts);
      if (rpc.from === 'rpc' && rpc.liqUsd != null) return rpc;
      return { liqUsd: null, ageMs, from: 'none' };
    }
    if (!(liqUsd != null && liqUsd > 0)) {
      return { liqUsd: null, ageMs, from: 'snapshot' };
    }
    return { liqUsd, ageMs, from: 'snapshot' };
  } catch (e) {
    log.warn({ err: (e as Error)?.message, pairAddress }, 'liq-watch snapshot read failed');
    return { liqUsd: null, ageMs: 0, from: 'none' };
  }
}

/**
 * Optional RPC fallback. Only called when snapshot path returned nothing
 * usable. Disabled by default (cfg.liqWatchRpcFallback === false).
 *
 * NOTE: full implementation requires vault_a/vault_b addresses to be
 * stamped onto OpenTrade (W7.5.1 spec). For W7.5 MVP this returns 'none'.
 */
async function maybeRpcFallback(args: LoadLiqArgs, ts: number): Promise<LoadLiqResult> {
  const { cfg } = args;
  if (!cfg.liqWatchRpcFallback) {
    return { liqUsd: null, ageMs: 0, from: 'none' };
  }
  // MVP: not implemented. See spec § "Знание вольтов".
  // Placeholder for W7.5.1: would qnCall('getMultipleAccounts', [[vaultA, vaultB], { encoding: 'jsonParsed' }], { feature: 'liq_watch', creditsPerCall: 30 }).
  void qnCall; // keep import alive for W7.5.1
  log.debug({ ts, msg: 'rpc fallback configured but vault addresses not stamped — pending W7.5.1' });
  return { liqUsd: null, ageMs: 0, from: 'none' };
}

export interface EvaluateArgs {
  cfg: PaperTraderConfig;
  entryLiqUsd: number;
  load: LoadLiqResult;
  consecutiveFailures: number;
  positionAgeMs: number;
}

export function evaluateLiqDrainState(args: EvaluateArgs): LiqWatchVerdict {
  const { cfg, entryLiqUsd, load, consecutiveFailures, positionAgeMs } = args;
  const ts = Date.now();
  if (positionAgeMs < cfg.liqWatchMinAgeMin * 60 * 1000) {
    return { kind: 'skipped', reason: 'pre-min-age', ts };
  }
  if (load.from === 'none' || load.liqUsd == null) {
    return { kind: 'pending', currentLiqUsd: null, consecutiveFailures, ageMs: null, ts };
  }
  if (!(entryLiqUsd > 0)) {
    return { kind: 'skipped', reason: 'no-entry-liq', ts };
  }
  const dropPct = +(((entryLiqUsd - load.liqUsd) / entryLiqUsd) * 100).toFixed(3);
  if (dropPct < cfg.liqWatchDrainPct) {
    return { kind: 'ok', currentLiqUsd: load.liqUsd, dropPct, ageMs: load.ageMs, from: load.from as 'snapshot' | 'rpc', ts };
  }
  // dropPct above threshold; require N consecutive failures before force-close.
  const next = consecutiveFailures + 1;
  if (next < cfg.liqWatchConsecutiveFailures) {
    return { kind: 'pending', currentLiqUsd: load.liqUsd, consecutiveFailures: next, ageMs: load.ageMs, ts };
  }
  return { kind: 'force-close', reason: 'LIQ_DRAIN', currentLiqUsd: load.liqUsd, dropPct, ageMs: load.ageMs, from: load.from as 'snapshot' | 'rpc', ts };
}
```
