# Spec W7.8 — On-chain simulate (аудит исполнимости, `feature: sim`)

**Статус:** **выполнено в коде** (продукт **1.3.0**, см. [`CHANGELOG.md`](../release/CHANGELOG.md)); нормативный текст спеки сохраняется как контракт/откат. Согласована с [`RELEASE_OPERATING_MODEL.md`](../release/RELEASE_OPERATING_MODEL.md) и [`PARALLEL_WORKFLOW.md`](../release/PARALLEL_WORKFLOW.md).

**Продукт:** `solana-alpha`, papertrader 2.0.  
**Зависит от:** **(m) W7.1** (`qn-client`, per-feature кредитомер), **W7.4** (Jupiter quote / цена для логики входа), **W6.3c** (executor, JSONL). **Рекомендуемый порядок внедрения:** после стабилизации W7.4; **W7.5 (liq)** ортогонален, но sim **логичнее** поверх непустой ликв-картины.

**Связанные артефакты:** `src/core/rpc/qn-feature-usage.ts` (ключ `'sim'`, дефолтный месячный бюджет), `src/core/rpc/qn-client.ts` (`qnCall`).

**Не путать с W7.6** — в репозитории **W7.6 = impulse confirm**; **W7.8 = simulateTransaction audit**.

---

## 1. Зачем (бизнес + live-перспектива)

- **W7.4** проверяет **HTTP quote** (Jupiter) относительно **снапшота** — это **не** гарантия, что **транзакция** с **теми же** (или согласованными) маршрутом/счетами **будет принята** сетью или **имеет смысл** при реальном исполнении.
- **W7.8** добавляет **аудит**: для **части** путей входа (sampling) строим **безопасную** (без private key) **транзакцию** (или **VersionedTransaction**), вызываем **`simulateTransaction`** по **тому же RPC** (`SA_RPC_HTTP_URL` / QuickNode), **учитываем кредиты** `feature: 'sim'`, пишем **результат** в **JSONL** (и опционально агрегат в heartbeat).
- **По умолчанию — не gate:** sim **не** блокирует `open` в v1 spec (только **teleops**; при желании v2 spec может добавить soft-flag **после** сбора статистики `paperSimAuditBlocksOpen` в отдельном change request).

**Ценность для будущего live (W8.0):** снизить класс **«quote ок, сеть/маршрут/accounts — нет»** и **«ожидание fill ≠ quote»** до **измеримого** разрыва, не заменяя **W7.2 / W7.4 / W7.5**.

---

## 2. Не-цели (явно out of scope v1)

- **Live-подпись** и **реальная** отправка (`sendTransaction`) — **вне** W7.8 (это W8.0+).
- **MEV, Jito bundles, приватные RPC** — вне v1.
- **Гарантия** совпадения sim и **будущего** live-фills на 100% — **нет**; sim = **лучшая** доступная **оценка** **до** кошелька.
- **Полный** parитет Jupiter swap API v6 на всех DEX-ветках day-one — допускается **поэтапно**: v1 may **OR** build tx через **Jupiter swap API (swap-instructions / legacy)** + sim **ИЛИ** only **synthetic** check с пометкой `simBuildKind: 'jupiter'|'skipped_unsupported'`.

- **Постоянные** `simulateTransaction` на **каждый** тик tracker — **запрещены** бюджетом и задержками; только **sampling** на **open-path** (или post-open **async** queue — **опционально** v1.1).

## 3. Инварианты безопасности (I* из release model, адаптация)

| ID | Инвариант |
|----|-----------|
| **W78-I1** | **Ни один** private key, **ни** `LIVE_WALLET_SECRET`, **ни** подпись пользователя. Только `simulateTransaction` + **read-only** данные. |
| **W78-I2** | **Все** кредиты QN через `qnCall({ feature: 'sim', creditsPerCall: N })`; **запрещён** сырой обход `qn-feature-usage`. |
| **W78-I3** | При `budget` / `rate` от `qnCall` — **degrade**: писать `sim: { kind: 'skipped', reason: 'qn_budget' | ... }` в JSONL, **не** **panic** main loop, **по умолчанию** **не** **блок** open. |
| **W78-I4** | **Latency cap:** wall-clock sim **per sampled open** ≤ `PAPER_SIM_MAX_WALL_MS` (default 4000..8000 ms); **при превышении** — `skipped: timeout` и **релиз** budget reservation если применим. |

## 4. Переменные окружения (обязательная схема)

| Env | Смысл | Default v1 |
|-----|--------|------------|
| `PAPER_SIM_AUDIT_ENABLED` | Глобальный выключатель | `0` |
| `PAPER_SIM_SAMPLE_PCT` | Семпл **0..100** (целое): какую долю **кандидатов на `open`**, **после** W7.4+gate, **пройти** sim | `0` = off; реком. старт `10` |
| `PAPER_SIM_MAX_WALL_MS` | Верхняя планка времени sim | `8000` |
| `PAPER_SIM_BUILD_TIMEOUT_MS` | fetch swap-instruction | `5000` |
| `PAPER_SIM_USE_JUPITER_BUILD` | `1` = строить IX через **публичный** Jupiter **swap**/**swap-instruction** (конкретный endpoint в коде) | `1` |
| `PAPER_SIM_CREDS_PER_CALL` | **Переопределение** `creditsPerCall` для `simulateTransaction` (или дефолт 30) | `30` |
| `PAPER_SIM_STRICT_BUDGET` | `1` = **не** делать sim, если `reserveQnFeatureCredits('sim')` не прошёл | `1` |

`QN_FEATURE_BUDGET_SIM` уже поддерживается в `qn-feature-usage.ts` (`DEFAULT_BUDGET.sim`).

## 5. Контракт JSONL

На событии `kind: 'open'` (или отдельное `kind: 'sim_audit'` **сразу** после `open` — **один** из вариантов, выбрать **один** в реализации и **зафиксировать** в CHANGELOG):

**Рекомендуемый минимум — поле на `open`:** `simAudit: SimAuditStamp | null`.

```ts
type SimAuditStamp =
  | { kind: 'skipped'; reason: 'disabled' | 'not_sampled' | 'no_build' | 'qn_budget' | 'timeout' | 'unimplemented_dex' | string; ts: number; wallMs?: number }
  | {
      kind: 'ok' | 'err';
      ts: number;
      wallMs: number;
      qnCredits: number;
      err?: { code: number; message: string } | null;
      /** solana sim result: units, logs snippet optional */
      unitsConsumed?: number | null;
      /** For analytics: notional or ratio vs paper, product-defined */
      notes?: string;
    };
```

**Запрещено:** **удалять** / **ломать** **существующие** поля `open`; только **add**. Старые строки без `simAudit` **валидны**.

## 6. Алгоритм (v1, логический)

1. **Pre:** `PAPER_SIM_AUDIT_ENABLED=0` → `simAudit: { kind: 'skipped', reason: 'disabled' }` или `null` **по** политике «явный vs null» (выбрать одно, default **null** = не писать для экономии места).
2. **Sampling:** применить **только** если `open` **уже** **решён** **к** **исполнению** в paper (после W7.4, safety, impulse policy и т.д.). `hash(openKey) % 100 < PAPER_SIM_SAMPLE_PCT` (или `crypto` stable hash) **или** round-robin — **указать** в **реализационных** **коммит**-ноте.
3. **Build unsigned tx** через Jupiter (если `PAPER_SIM_USE_JUPITER_BUILD=1`): `input/output/slippage` **согласовать** **с** W7.4 notional/мятой **(или** **явно** **лог** **«mismatch intentionally»**).
4. **`qnCall('simulateTransaction', [txBase64, { replaceRecent... }], { feature: 'sim', creditsPerCall, timeoutMs })`**. Парсинг **err** / **ок**.
5. **Персист** в `simAudit` на `open` (или отдельной строке **с** `refMint` + `ts`).
6. **Main loop:** sim **async** after open committed **to JSONL** **допустим** (не блокирует `open` **дольше** cap); если async — `simAudit` обновляется **в** **следующем** `append` **событии** **или** **re-open** file policy — **проще** **sync** **в** **v1** **только** если `wall` **&lt;** 2s, иначе **v1.1** async.

**v1 рекомендация:** **синхронно** **только** **если** `PAPER_SIM_MAX_WALL_MS` **жёсткий** и sempahore **1** per process **избегаем** burst; иначе **микро-queue** in-process.

## 8. Ветвление, интеграция, релиз (регламент)

Согласно [`PARALLEL_WORKFLOW.md`](../release/PARALLEL_WORKFLOW.md):

1. **Исполнитель** создаёт ветку от актуального `v2`, например **`task/w7-8-sim-audit`**.
2. Коммиты **только** в `task/*`; **прямой** push в `v2` **по** умолчанию **не** **обязателен** (или по договорённости).
3. **Интегратор** (или вы): merge в `v2` → `npm run typecheck` / тесты → **один** **бамп** `docs/strategy/release/VERSION` (MINOR) + блок **CHANGELOG** с:
   - **что** (W7.8 sim audit, sampling, JSONL);
   - **как** откатить (revert + `PAPER_SIM_AUDIT_ENABLED=0` on server);
   - **тег** `sa-alpha-X.Y.Z` по **желанию** ([§4.5](release/RELEASE_OPERATING_MODEL.md#45-связь-с-git-тегами)).
4. **Деплой** на сервер **один** `git reset --hard` на тот **SHA** + `pm2 flush` / `restart` paper-приложения **per** [server autodeploy rule] — **salpha**, не ломая другие продукты.
5. **docs/platform** **не** трогать — **не** **требуется** platform `VERSION` bump.

**Cross-product:** только `solana-alpha` — **нет** `products.yaml` / platform changelog **если** **не** **трогали** `docs/platform/**`.

## 9. Stage → Prod: что это значит **у нас** (один VPS)

**«Stage»** **не** обязан = **второй** **сервер**.


| Модель | Суть | Сложность **сейчас** |
|--------|------|------------------------|
| **A. Feature flag (реком. v1)** | `PAPER_SIM_AUDIT_ENABLED=0` на **prod** до **готовности**; первый **выкат** с `SAMPLE_PCT=5-10` **только** на **одной** **стратегии** (напр. `pt1-dno`) **в** `ecosystem.config.cjs` **+** `pm2 restart` **one** | **Низкая** — только env-переопределение per-app |
| **B. «Stage» = тот же бинарь, другое** `PAPER_TRADES_PATH` | **Дублирующий** **PM2** `pt1-oscar-simlab` (пример) с `SIM_*` on и **другой** **JSONL** — **не** смешивать **с** **боевым** журналом | **Средняя** — новая **app** в **ecosystem** + **диск** + **дисциплина** |
| **C. Второй VPS** | **Настоящий** pre-prod | **Высокая** — **не** **требуется** **для** W7.8 v1 |

**Рекомендация W7.8 v1:** **A** (canary on **one** strategy, **низкий** `SAMPLE_PCT` **→** **raise** after 7d **or** no anomalies).

**Сложность внедрения** **сейчас** **(код):** **средняя** — **потому что** **нужно** **согласовать** **Jupiter** **build** (unsigned) **+** sim **+** **не** **сломать** **latency** **и** **бюджет**; **спек** **(этот** **документ) —** **низкая**; **код+деплой** **—** **1–3** **дня** **инженерно** **(оценка)**.

## 10. Rollback (операции)

1. `PAPER_SIM_AUDIT_ENABLED=0` (и `PAPER_SIM_SAMPLE_PCT=0`) **на** **всех** **pt1-***;
2. `pm2 ... --update-env` **+** `flush` **per** runbook;
3. при **необходимости** `git revert` **merge** **commit** W7.8;
4. **журналы** **с** `simAudit` **историчны** — **не** **трогать** **(replay** **должен** **игнорировать** **неизвестные** **поля** **или** **tolerate** `simAudit`).
