# Spec W7.4.1 — Jupiter quote retries + circuit breaker

**Назначение:** снизить долю ложных `skipped` на транзитных сбоях lite-api Jupiter и ограничить шторм запросов при деградации API.

**Реализация (код):**

- `src/papertrader/pricing/jupiter-quote-resilience.ts` — окно событий, порог trip, `circuit-open`.
- `src/papertrader/pricing/price-verify.ts` — backoff-цикл вокруг `jupiterQuoteBuyPriceUsdOnce` / `jupiterQuoteSellPriceUsdOnce`; `fetchJupiterBuyQuoteResponse` (W7.8 sim-audit).
- `quoteResilienceFromPaperCfg` в `src/papertrader/config.ts` — собирает опции из env (если и retries и circuit выкл. → поведение как до W7.4.1).
- Импульс W7.6: `impulse-confirm.ts` передаёт тот же resilience, что paper-config.

**Политика:**

- Повторяются только `skipped` с причинами: `http-error`, `timeout`, `fetch-fail`, `parse-error`.
- Успех транспорта: вердикт `ok` или `blocked` (в т.ч. `no-route`, slip/impact) — в статистику breaker как **успех**.
- Circuit: скользящее окно `PAPER_PRICE_VERIFY_CIRCUIT_WINDOW_MS` (дефолт 30 мин); если после минимума выборки `PAPER_PRICE_VERIFY_CIRCUIT_MIN_ATTEMPTS` доля транспортных неудач **строго выше** `PAPER_PRICE_VERIFY_CIRCUIT_SKIP_RATE_PCT` — breaker открывается на `PAPER_PRICE_VERIFY_CIRCUIT_COOLDOWN_MS`. Пока открыт, новые quote возвращают `skipped` / `reason: circuit-open` (или `null` в raw fetch для sim-audit).

**ENV (префикс как у W7.4):**

| Переменная | Дефолт (schema) |
|------------|-----------------|
| `PAPER_PRICE_VERIFY_QUOTE_RETRIES_ENABLED` | `1` |
| `PAPER_PRICE_VERIFY_QUOTE_MAX_ATTEMPTS` | `3` |
| `PAPER_PRICE_VERIFY_QUOTE_RETRY_BACKOFF_MS` | `300` |
| `PAPER_PRICE_VERIFY_CIRCUIT_ENABLED` | `1` |
| `PAPER_PRICE_VERIFY_CIRCUIT_WINDOW_MS` | `1800000` |
| `PAPER_PRICE_VERIFY_CIRCUIT_SKIP_RATE_PCT` | `10` |
| `PAPER_PRICE_VERIFY_CIRCUIT_MIN_ATTEMPTS` | `12` |
| `PAPER_PRICE_VERIFY_CIRCUIT_COOLDOWN_MS` | `90000` |

**Откат:** `PAPER_PRICE_VERIFY_QUOTE_RETRIES_ENABLED=0` и `PAPER_PRICE_VERIFY_CIRCUIT_ENABLED=0` — одиночный запрос, без breaker (как до этого шага).
