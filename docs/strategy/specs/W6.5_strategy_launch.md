# Spec W6.5 — Запуск трёх dip-стратегий в paper (DipRunner / Oscar / Dno)

**Назначение spec:** для агента-исполнителя (cheap LLM). Выполнить последовательно, не отступая ни от одного шага.

**Контекст продукта:** `solana-alpha`. Репо: `c:/Users/cente/Ideas/solana-alpha`. Ветка `v2`. Текущий HEAD после W6.4 + INDEX-патч: `4108c3b`.

**Время выполнения:** ~2.5 часа. Это «launch»-этап: реальной кодовой работы мало (узкий патч `whale-analysis.ts` + `paper2-healthcheck.mjs`), основное — конфигурация трёх PM2-апов с точными ENV-блоками из salvage.

## Цель

После W6.4 на VPS работает один **smoke**-app `pt1-dip-executor-test` со «слабыми» фильтрами и whale OFF — это было для DoD W6.3c, не торговля. W6.5 заменяет его на **три боевые dip-стратегии**:

1. **`pt1-diprunner`** (Deep Runner / DipRunner) — быстрый дип-скальп на свежих post-migration монетах (age 0+), DCA aggressive (-7% / -15%), ladder TP 5/10/15%, killstop −22%, timeout 1.5h.
2. **`pt1-oscar`** — длинная лестница (30/60/100%), мягкий DCA (-12%, killstop −25%), TP=3x, timeout 12h. Тот же вход, что у DipRunner.
3. **`pt1-dno`** — «дно» на зрелых монетах (age 180–4320 мин = 3–72 ч), без DCA, ладдер 5/10/20/30%, SL=0.9 (-10%), TP=1.5x, timeout 1h.

Все три используют **whale-analysis включённой**, но **без `whaleRequireTrigger`** (логируем whale-контекст в каждый `eval`/`open`, но не режем поток на старте — иначе на тонкой базе сделок не будет; гейт включит W6.6 на основе данных лаба после ≥7 дней работы).

Дополнительно W6.5 добавляет **узкий код-патч** (по одному вопросу пользователя «у кита может остаться много и он ещё дольёт»):

4. **`whaleSilenceMinAfterLastSell`** (env `PAPER_DIP_WHALE_SILENCE_MIN`, default `0` = OFF) — `wallet` из `large_sells` НЕ становится `capitulator`, если с момента **его** последнего sell-свопа по этому минту прошло **меньше** N минут. Логика: «слил большое и затих ≥ N мин» = капитуляция; «слил большое и продолжает дробить» = ждём.

5. **`paper2-healthcheck.mjs`** — переключаем с mtime-based на **heartbeat-based** проверку: алерт если в `*.jsonl` **нет** строки `kind=heartbeat` свежее `PAPER2_MAX_STALE_MIN` минут. Это убирает ложно-stale алёрты на боевых стратегиях, у которых `eval`-поток может быть редким, но heartbeat пишется каждые 30s.

После W6.5 на VPS: **13 PM2 процессов** (10 из W6.0/W6.1/W6.4 + 3 paper). Старого `pt1-dip-executor-test` нет.

## Шаг 3 — Patch: `scripts-tmp/paper2-healthcheck.mjs` — heartbeat-based проверка

**Заменить файл целиком** на:

```javascript
import 'dotenv/config';
import fs from 'node:fs';
import path from 'node:path';
import { sendTagged } from '../scripts/lib/telegram.mjs';

const PAPER2_DIR = process.env.PAPER2_DIR || '/opt/solana-alpha/data/paper2';
const MAX_STALE_MIN = Number(process.env.PAPER2_MAX_STALE_MIN || 20);
const TAIL_LINES = Number(process.env.PAPER2_HEALTH_TAIL_LINES || 1500);
const BOT_TOKEN = process.env.TELEGRAM_BOT_TOKEN || '';
const CHAT_ID = process.env.TELEGRAM_CHAT_ID || '';
const HEALTH_TELEGRAM_ON_ALERT = process.env.PAPER2_HEALTH_TELEGRAM_ON_ALERT !== '0';
const HEALTH_TELEGRAM_ON_OK = process.env.PAPER2_HEALTH_TELEGRAM_ON_OK === '1';

function listStrategyFiles() {
  if (!fs.existsSync(PAPER2_DIR)) return [];
  return fs
    .readdirSync(PAPER2_DIR)
    .filter((f) => f.endsWith('.jsonl'))
    .map((f) => path.join(PAPER2_DIR, f))
    .sort();
}

function tailLastHeartbeatTs(filePath) {
  if (!fs.existsSync(filePath)) return null;
  let buf;
  try {
    buf = fs.readFileSync(filePath, 'utf-8');
  } catch {
    return null;
  }
  const lines = buf.split('\n');
  const start = Math.max(0, lines.length - TAIL_LINES);
  for (let i = lines.length - 1; i >= start; i--) {
    const ln = lines[i];
    if (!ln) continue;
    if (!ln.includes('"heartbeat"')) continue;
    try {
      const j = JSON.parse(ln);
      if (j && j.kind === 'heartbeat') {
        if (typeof j.ts === 'number') return j.ts;
        const fileTs = fs.statSync(filePath).mtimeMs;
        return fileTs || null;
      }
    } catch {
      // skip
    }
  }
  return null;
}

async function sendTelegram(text) {
  if (!HEALTH_TELEGRAM_ON_ALERT && !HEALTH_TELEGRAM_ON_OK) return;
  if (!BOT_TOKEN || !CHAT_ID) return;
  await sendTagged('HEALTH', 'platform', text);
}

async function main() {
  const now = Date.now();
  const files = listStrategyFiles();
  if (!files.length) {
    const msg = `paper2 health: no strategy files in ${PAPER2_DIR}`;
    console.log(msg);
    if (HEALTH_TELEGRAM_ON_ALERT) await sendTelegram(msg);
    process.exitCode = 1;
    return;
  }

  const stale = [];
  const ok = [];
  for (const fp of files) {
    const id = path.basename(fp, '.jsonl');
    const hbTs = tailLastHeartbeatTs(fp);
    if (!hbTs) {
      stale.push({ id, ageMin: null, reason: 'no_heartbeat_in_tail' });
      continue;
    }
    const ageMin = (now - hbTs) / 60000;
    if (ageMin > MAX_STALE_MIN) stale.push({ id, ageMin, reason: 'heartbeat_stale' });
    else ok.push({ id, ageMin });
  }

  const summary = {
    now: new Date(now).toISOString(),
    paper2Dir: PAPER2_DIR,
    maxStaleMin: MAX_STALE_MIN,
    metric: 'heartbeat',
    total: files.length,
    ok: ok.length,
    stale: stale.length,
  };
  console.log(JSON.stringify(summary, null, 2));

  if (stale.length > 0) {
    const details = stale
      .map((s) =>
        `- ${s.id}: ${s.ageMin === null ? s.reason : `${s.ageMin.toFixed(1)}m stale (${s.reason})`}`,
      )
      .join('\n');
    const text =
      `paper2 health ALERT (heartbeat-based)\n` +
      `stale=${stale.length}/${files.length} (threshold ${MAX_STALE_MIN}m)\n\n${details}`;
    if (HEALTH_TELEGRAM_ON_ALERT) await sendTelegram(text.slice(0, 3900));
    process.exitCode = 1;
    return;
  }

  const okTop = ok
    .sort((a, b) => a.ageMin - b.ageMin)
    .slice(0, 8)
    .map((s) => `${s.id}:${s.ageMin.toFixed(1)}m`)
    .join(', ');
  if (HEALTH_TELEGRAM_ON_OK) {
    await sendTelegram(`paper2 health OK (heartbeat)\n${ok.length}/${files.length} active\n${okTop}`.slice(0, 3900));
  } else {
    console.log(`paper2 health OK (telegram suppressed): ${ok.length}/${files.length} active`);
  }
}

main().catch((e) => {
  console.error(e);
  process.exit(1);
});
```

Изменения по сравнению с предыдущей версией:
- метрика свежести = последний `kind=heartbeat` в tail последних `TAIL_LINES` строк (default 1500), а не `mtime`;
- если `heartbeat.ts` отсутствует — fallback на `mtime` файла (для совместимости со старыми JSONL без `ts` в heartbeat);
- в summary появилось поле `metric: 'heartbeat'`;
- порог `PAPER2_MAX_STALE_MIN` остаётся тем же ENV-именем (cron не меняем).
