## Зачем нужна эта spec

Пользователь явно сформулировал две проблемы:

1. **Скам-токены проходят entry-фильтры.** На стратегиях `pt1-diprunner`/`pt1-oscar`/`pt1-dno` сейчас работает только snapshot-фильтрация (объём, holders, дип). Нет проверки on-chain, что токен не «honeypot» (mint-authority не отозван — можно выпустить ещё; freeze-authority не отозван — можно заморозить кошелёк; top-1 holder держит >40% supply — типичный ралли-дамп).
2. **`mcap n/a` в audit-timeline.** Дашборд `/papertrader2` теперь умеет показывать полный timeline сделки (open → DCA → ladder TP → close), но в текущих PT2-стратегиях `metricType === 'price'` (отслеживается цена токена, не mcap), поэтому в каждой строке timeline пишется `mcap n/a`. Это убивает аудит-ценность («во сколько выкупил?», «во сколько лесенкой продал?»). Нужна явная USD-mcap записываемая в каждое событие jsonl, чтобы строка timeline выглядела ровно как просил пользователь:

   ```text
   01.05, 06:42:19   Ladder (take profit) · sell 30% of remaining · mcap $770.1k · +$12.40
   ```

W7.2 закрывает оба пункта одним инкрементом, **строго в рамках paper-trader 2.0** (без размораживания `sa-stream`/`sa-parser`/`sa-atlas`).

## Цель

После W7.2:

1. **Safety Check (pre-entry).** Каждая стратегия PT2 (`pt1-diprunner` / `pt1-oscar` / `pt1-dno`), у которой включён `PAPER_SAFETY_CHECK_ENABLED=1`, перед `appendEvent({kind:'open'})` делает 1 батчевой запрос QN RPC через **уже существующий** `qnCall()` wrapper из `src/core/rpc/qn-client.ts` с `feature: 'safety'`. Запрос забирает:
   - `getAccountInfo(mint, { encoding: 'jsonParsed' })` — для `mint_authority`, `freeze_authority`, `decimals`, `supply`.
   - `getTokenLargestAccounts(mint)` — top-20 token-accounts mint'а.

   На основании результата строится вердикт `SafetyVerdict`:
   - `ok: false` если `mint_authority !== null` или `freeze_authority !== null` (для **всех** mint'ов);
   - `ok: false` если `top_holder_pct > PAPER_SAFETY_TOP_HOLDER_MAX_PCT` (по умолчанию `40`) **и** mint **не AMM** (см. ниже LP-исключение);
   - иначе `ok: true`.

   Если `ok: false` — позиция **не открывается**; в jsonl пишется `eval-skip-open` с `reason: 'safety:<csv>'`. Если `ok: true` — позиция открывается, и в `open`-событие добавляется поле `safety: SafetyVerdict` (для будущего пост-морт-разбора).

   Если `qnCall()` вернул `ok: false` (`reason ∈ {budget, rate, http, timeout, rpc_error}`) — safety **пропускается** (fail-open: торговля важнее, чем 100%-кавер safety). В jsonl пишется `safety: { skipped: '<reason>' }` в open-событии — это аудит-след, что safety не сработал.

2. **Live Market Cap в timeline.** Каждое событие `dca_add` / `partial_sell` / `close` в jsonl получает новое поле `mcUsdLive: number | null`. Источник:
   - сначала `fetchLatestSnapshotMcap(mint, ot.source)` — выбирает свежайшую строку из `<source>_pair_snapshots`, берёт `market_cap_usd`, иначе `fdv_usd`;
   - если snapshot пустой (пампы без AMM) — `fetchPumpfunMc(mint).mc` (уже существует в `pricing.ts`);
   - если оба пустые — `null`.
   Кэш в RAM на `PAPER_LIVE_MCAP_TTL_MS` (default `30000` мс) per-mint, чтобы не молотить snapshot DB / pump.fun frontend на каждый tick.

3. **Dashboard timeline** (`scripts-tmp/dashboard-server.ts`) подхватывает новое поле без новых API-схем: в `buildTimelineEvent` для `dca_add` / `partial_sell` / `close` сначала смотрим `e.mcUsdLive`, и только если оно отсутствует — fallback на текущую логику (`marketPrice` если `metricType==='mc'`, иначе `null`).

После W7.2 в дашборде `/papertrader2` строка timeline под раскрытой позицией будет выглядеть **точно** как в приведённом скриншоте. Старые позиции (открытые до деплоя) останутся с `mcap n/a` — это нормально, аудит улучшается монотонно.

## Шаг 1 — Live Market Cap helper (`src/papertrader/pricing.ts`)

Добавь в конец файла два экспорта (НЕ удаляй и НЕ модифицируй существующие функции):

```ts
// ---------------------------------------------------------
// Live USD market cap helper (W7.2): used by tracker to stamp
// every dca_add / partial_sell / close jsonl event with the
// observed mcap at the moment of the event.
// ---------------------------------------------------------

export async function fetchLatestSnapshotMcap(
  mint: string,
  source?: 'raydium' | 'meteora' | 'orca' | 'moonshot',
): Promise<number | null> {
  const tables: string[] = source
    ? [`${source}_pair_snapshots`]
    : ['raydium_pair_snapshots', 'meteora_pair_snapshots', 'orca_pair_snapshots', 'moonshot_pair_snapshots'];
  const safeMint = mint.replace(/'/g, "''");
  for (const t of tables) {
    const r = await db.execute(dsql.raw(`
      SELECT market_cap_usd, fdv_usd
      FROM ${t}
      WHERE base_mint = '${safeMint}'
      ORDER BY ts DESC
      LIMIT 1
    `));
    const rows = r as unknown as Array<{ market_cap_usd: number | string | null; fdv_usd: number | string | null }>;
    const mc = Number(rows[0]?.market_cap_usd ?? 0);
    if (mc > 0) return mc;
    const fdv = Number(rows[0]?.fdv_usd ?? 0);
    if (fdv > 0) return fdv;
  }
  return null;
}

const _liveMcCache = new Map<string, { mc: number; ts: number }>();
function liveMcTtlMs(): number {
  const v = Number(process.env.PAPER_LIVE_MCAP_TTL_MS || 30_000);
  return Number.isFinite(v) && v > 0 ? Math.floor(v) : 30_000;
}

/**
 * Best-effort live USD market cap.
 * Order: pair_snapshots (free) → pump.fun frontend (free) → null.
 * Cached per mint for PAPER_LIVE_MCAP_TTL_MS (default 30s) to keep external
 * call rate sane on a tight tracker tick.
 */
export async function getLiveMcUsd(
  mint: string,
  source?: 'raydium' | 'meteora' | 'orca' | 'moonshot',
): Promise<number | null> {
  const cached = _liveMcCache.get(mint);
  if (cached && Date.now() - cached.ts < liveMcTtlMs()) return cached.mc || null;
  let mc: number | null = null;
  try {
    mc = await fetchLatestSnapshotMcap(mint, source);
  } catch {
    /* swallow — pricing is best-effort */
  }
  if (mc == null || !(mc > 0)) {
    try {
      const p = await fetchPumpfunMc(mint);
      if (p && p.mc > 0) mc = p.mc;
    } catch {
      /* swallow */
    }
  }
  _liveMcCache.set(mint, { mc: mc && mc > 0 ? mc : 0, ts: Date.now() });
  return mc && mc > 0 ? mc : null;
}
```

## Шаг 4 — Safety Check модуль

Создай новый файл `src/papertrader/safety/index.ts`:

```ts
/**
 * W7.2 — Pre-entry safety check.
 *
 * Single batched QuickNode RPC call (feature: 'safety', ~60 credits) that
 * fetches mint authorities + supply + top-20 token accounts, and returns
 * a verdict the executor can use to skip openly bad mints.
 *
 * Fail-open by design: if QN budget exhausted / RPC failed → skipped.
 */
import { qnBatchCall } from '../../core/rpc/qn-client.js';
import { child } from '../../core/logger.js';
import type { SafetyVerdict } from '../types.js';

const log = child('papertrader-safety');

type AccountInfoParsed = {
  value: {
    data: {
      parsed: {
        info: {
          decimals: number;
          freezeAuthority: string | null;
          mintAuthority: string | null;
          supply: string;          // u64 as string
        };
      };
    };
  } | null;
};

type LargestAccountsResult = {
  value: Array<{ address: string; amount: string; uiAmount: number }>;
};

function topHolderPct(largest: LargestAccountsResult['value'], totalSupply: string): number | null {
  if (!largest.length || !totalSupply) return null;
  const total = Number(totalSupply);
  if (!Number.isFinite(total) || total <= 0) return null;
  const top = Number(largest[0].amount);
  if (!Number.isFinite(top) || top <= 0) return null;
  return (top / total) * 100;
}

export type SafetyOptions = {
  topHolderMaxPct: number;
  requireMintAuthorityNull: boolean;
  requireFreezeAuthorityNull: boolean;
  /** AMM mints typically have a pool vault as top-1; skip top-1 check for them. */
  treatAsAmm: boolean;
  timeoutMs: number;
};

export type SafetyOutcome =
  | { kind: 'verdict'; verdict: SafetyVerdict }
  | { kind: 'skipped'; reason: string };

export async function evaluateMintSafety(
  mint: string,
  opts: SafetyOptions,
): Promise<SafetyOutcome> {
  const res = await qnBatchCall<AccountInfoParsed | LargestAccountsResult>(
    [
      { method: 'getAccountInfo', params: [mint, { encoding: 'jsonParsed' }] },
      { method: 'getTokenLargestAccounts', params: [mint] },
    ],
    { feature: 'safety', creditsPerCall: 30, timeoutMs: opts.timeoutMs },
  );
  if (!res.ok) {
    log.debug({ mint, reason: res.reason }, 'safety: qnCall failed, skipping');
    return { kind: 'skipped', reason: res.reason };
  }
  const [acc, largest] = res.value as [AccountInfoParsed, LargestAccountsResult];
  const info = acc?.value?.data?.parsed?.info;
  if (!info) {
    return { kind: 'skipped', reason: 'mint_account_missing' };
  }
  const mintAuthority = info.mintAuthority;
  const freezeAuthority = info.freezeAuthority;
  const decimals = Number(info.decimals);
  const supply = String(info.supply);
  const tHolder = topHolderPct(largest?.value || [], supply);
  const reasons: string[] = [];
  if (opts.requireMintAuthorityNull && mintAuthority !== null) {
    reasons.push(`mint_authority=${mintAuthority.slice(0, 8)}…`);
  }
  if (opts.requireFreezeAuthorityNull && freezeAuthority !== null) {
    reasons.push(`freeze_authority=${freezeAuthority.slice(0, 8)}…`);
  }
  if (!opts.treatAsAmm && tHolder != null && tHolder > opts.topHolderMaxPct) {
    reasons.push(`top1=${tHolder.toFixed(1)}%`);
  }
  const verdict: SafetyVerdict = {
    ok: reasons.length === 0,
    reasons,
    mint_authority: mintAuthority,
    freeze_authority: freezeAuthority,
    top_holder_pct: tHolder,
    decimals: Number.isFinite(decimals) ? decimals : null,
    supply,
    ts: Date.now(),
  };
  return { kind: 'verdict', verdict };
}
```
