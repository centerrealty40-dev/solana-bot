# Idealized Oscar stack — сводная спецификация (Paper ∥ Live)

Документ фиксирует **единый целевой замысел** стратегии dip/post-lane (ветка Oscar) и **организацию двух параллельных процессов** для пост-анализа и развития продукта:

| Процесс | PM2 app | Роль |
|---------|---------|------|
| **Paper Oscar** | `pt1-oscar` | Экспериментальный контур: полный журнал, **классы пути до входа** (`down` / `sideways` / `up` / `unknown`) задают **профиль выхода** (сетка, kill, ограничение ступеней). Нотионал **$100**. |
| **Live Oscar** | `live-oscar` | Боевой контур: **без классов пути** на входе для вилки выходов; после входа выходная политика — **два режима A / B** по факту **усреднения** (DCA по правилам трекера и/или исполненная вторая нога scale-in). Реальные лимиты и исполнение Phase 4–7. |

**Общее для обоих (не разносим без отдельного решения):** профили PM2 как в §1; лейны и вселенная §2; усиленные пороги ликвидности/объёма §3; политика cooldown §4; дип-окно, импульс, recovery veto §5; пост-импульс W7.6 §6; рекомендации контрфакта §7.2 (по мере имплементации — **оба** контура).

Источник правды по env для PM2 — **`ecosystem.config.cjs`**; при расхождении с этим файлом править спецификацию или конфиг.

## 0. Матрица: что одинаково, что разводится

| Область | Paper Oscar | Live Oscar |
|---------|-------------|------------|
| Лейны, дип, импульс, recovery, W7.6 | ✓ общие принципы и целевые env | ✓ то же |
| Liq / vol / holders / guards (§3) | ✓ целевые пороги **200k / 20k** (внедрение — см. §14) | ✓ то же |
| Loss-only cooldown 12h (§4.2) | ✓ после внедрения | ✓ после внедрения |
| **Класс пути до входа** (`tp-regime`) | ✓ **включает вилку выходов** по классу | ✗ **не использует** классы для выходной логики |
| **Вилка выходов после входа** | Зависит от **класса монеты** (и overrides сетки/kill) | Зависит только от **режима A vs B** (было ли усреднение по правилам продукта) |
| Нотионал, DCA-конструкция | §8.1 ($100, DCA уровни paper) | §8.2 ($40, двухногий вход, см. live spec) |
| TP / trail / timeout | §9.1 + overrides класса | §9.2 режимы **A / B** (параметры отдельно от paper) |

## 4. Cooldown и частота входов

### 4.1 Уже в коде (entry → entry, тот же mint) — **оба**

| Параметр | Env | Oscar |
|----------|-----|-------|
| Базовый cooldown | `PAPER_DIP_COOLDOWN_MIN` | **30** мин |
| «Скальп» после predictable DCA | `PAPER_DIP_COOLDOWN_MIN_SCALP` | **20** мин |

Логика: `dip-clones.ts` сравнивает время с **последним entry** по mint (`lastEntryTsByMintMap`).

### 4.2 Утверждённая операционная политика: loss-only **12 часов** после выхода — **оба**

**Решение:** после **убыточного** закрытия позиции на mint следующий вход на этот mint разрешён не раньше чем через **12 часов** от времени **exit** предыдущей убыточной сделки; после прибыльного закрытия дополнительная пауза по этому правилу **не требуется**.

**Статус внедрения:** отдельного env и ветки в `dip-clones.ts` под «exit→next entry после минуса» в актуальной кодовой базе **нет** — действует только минутный cooldown по **entry**. Реализация: новый флаг (например `PAPER_DIP_LOSS_EXIT_COOLDOWN_HOURS=12`) + учёт `exitTs` и знака PnL по журналу/state **в общем gate для paper и live**.

## 7. Анти-«перегретые» монеты и режим пути до входа

### 7.1 Развод Paper vs Live (нормативно)

| Вопрос | Paper Oscar (`pt1-oscar`) | Live Oscar (`live-oscar`) |
|--------|-------------------------|---------------------------|
| Классификация пути до входа | **Да:** торгует выходную стратегию с опорой на **три содержательных класса** ценового пути в окне до входа (`down` / `sideways` / `up`; плюс `unknown` при недостатке данных) через механику **`tp-regime`** / `stampTpRegimeAtHistoricalEntry` в анализе и целевой включённости **`PAPER_TP_REGIME_ENABLED=1`** в paper. | **Нет:** **не вводит** эту классификацию как переключатель выходов; прежняя единая база + переход на **режимы A/B** по факту усреднения (§8.2, §9.2). |
| Смысл классов для продукта | Задаёт **разные overrides** сетки TP / kill (см. `tp-regime.ts`): например жёстче в `down`, уже в `sideways`, без overrides в `up`. | Классы **не задают** deny на вход и **не дублируют** exit-ветку; входные «красные флаги» — из §3–§6 и §7.2 после имплементации. |

**Замечание:** метка «режим пути» **не является** сама по себе запретом покупки; запреты — только явные гейты (liq/vol, контрфакт §7.2, и т.д.). Это согласовано с пост-анализом: даун-класс часто совпадает с зоной edge dip-стратегии.

### 7.2 Реализация tp-regime в paper (когда включено)

Включается только если **`PAPER_TP_REGIME_ENABLED=1`**. В базовом `pt1-oscar` в `ecosystem.config.cjs` флаг может быть **не выставлен** — тогда режим по умолчанию **выкл.** до решения о rollout.

Когда включено:

- Окно классификации: **`tpRegimeLookbackMin`** (default **720** мин в `config.ts`).
- Классы: **`down`**, **`up`**, **`sideways`**, **`unknown`** по порогам `tpRegimeDownNetPct` / `tpRegimeUpNetPct` / sideways (defaults в `config.ts`).
- Переопределения сетки TP (фактическая семантика — в `overridesForRegime`):
  - **`down`:** по сути одна ступень (`gridMaxRungs: 1`, `gridSellFraction: 1`), опционально жёсткий kill **`PAPER_TP_REGIME_DOWN_DCA_KILLSTOP`** (например **-0.07** при включённом режиме).
  - **`sideways`:** например `gridMaxRungs: 2`.
  - **`up` / `unknown`:** без переопределений.

### 7.3 Эвристики из контрфакта (рекомендации; к **обоим** при имплементации)

По анализу пути до входа (`paper2-counterfactual-gates.ts` / `paper2-loss-attribution-deep-dive.ts`) зафиксированы **идеализированные фильтры**:

| Идея | Суть | Статус в коде discovery |
|------|------|---------------------------|
| Исключить «uptrend» перед входом | Режим пути `pre_entry_uptrend` — хуже статистика | **Нет** отдельного deny-кода в `dip-clones`; нужна имплементация по PG-пути |
| Не входить у верхней границы диапазона | `entry_hi ≥ 72%` от 48h range — хуже | Аналогично |
| Комбо склон + высота в диапазоне | `preSlope > 0.0008` и `entry_hi ≥ 65%` | Аналогично |
| Исключить «bounce chase» | Отскок от раннего дна + высокая позиция в range | Аналогично |

Эти пункты — **часть idealized spec** для следующих итераций gate’ов; текущий прод-фильтр опирается на пост-lane метрики, дип, киты, holders, impulse confirm, price verify.

## 8. DCA и размер позиции

### 8.1 Paper Oscar (`pt1-oscar`)

| Параметр | Env | Значение |
|----------|-----|----------|
| Нотионал | `PAPER_POSITION_USD` | **100** |
| DCA уровни | `PAPER_DCA_LEVELS` | **`-7:0.45`** (−7% от средней, докупка **45%** от нотионала первой ноги) |
| Kill-stop | `PAPER_DCA_KILLSTOP` | **-0.14** (−14% к средней) |
| Kill при режиме `down` (если включён TP regime) | `PAPER_TP_REGIME_DOWN_DCA_KILLSTOP` | **-0.07** (пример) |

**Выходная логика:** тейк-профит (сетка), усреднение, kill и трейл **зависят от класса пути** при включённом **`PAPER_TP_REGIME_ENABLED`** (§7). Без включения режима — единый профиль сетки из §9.1.

### 8.2 Live Oscar (`live-oscar`)

| Параметр | Env | Значение |
|----------|-----|----------|
| Нотионал (paper-слой = лимит сделки) | `PAPER_POSITION_USD` | **40** (= `LIVE_MAX_POSITION_USD`) |
| Первая нога | `PAPER_ENTRY_FIRST_LEG_FRACTION` | **0.7** (~70% / ~30% двухногий вход через scale-in) |
| Задержка второй ноги | `LIVE_ENTRY_SCALE_IN_DELAY_MS` | **5000** (трекер сначала оценивает частичный TP; при срабатывании ступени сетки вторая нога не исполняется) |
| Коридор Jupiter ко 2-й ноге | `LIVE_ENTRY_SCALE_IN_CORRIDOR_UP_PCT` / `DOWN` | **+5** / **−7** (% к якорю первой ноги) |
| DCA между ногами | `PAPER_DCA_LEVELS` | **пусто** — запланированная вторая нога по коридору Jupiter (и отдельно политика трекера для классического DCA, если включена в конфиге live) |
| Kill-stop (база) | `PAPER_DCA_KILLSTOP` | **-0.05** |

**Два режима выхода (нормативная цель продукта):**

| Режим | Условие (операционное определение) | Смысл |
|-------|-------------------------------------|--------|
| **A** | После входа **не было** исполненного усреднения по правилам продукта: ни шага **`dca_add`** по уровням DCA, ни исполненной **второй ноги scale-in**, увеличивающей экспозицию до закрытия. | Профиль выхода «полная лестница» / более широкий тейк-профиль и согласованные trail/timeout (**параметры зафиксировать в env live**, отдельно от paper). |
| **B** | Было **хотя бы одно** усреднение: срабатывание DCA по **`PAPER_DCA_LEVELS`** или успешная вторая нога **scale-in** (увеличение позиции после первой покупки). | Профиль «после боли» — более жёсткая сетка / раньше фиксация / согласованный kill (**параметры зафиксировать в env live**). |

**Статус внедрения:** переключение трекера live по **A/B после факта** должно быть явно реализовано (см. §15); текущее поведение может совпадать с единым профилем до rollout.

## 9. Take-profit, трейл, тайм-ауты

**Нормативно:** параметры **разные** у Paper Oscar и Live Oscar: paper опирается на **класс пути** + базовую сетку; live — на **режим A или B** после входа, без tp-regime классов.

### 9.1 Paper Oscar — сетка TP + ladder_retrace

| Параметр | Env | Значение (референс ecosystem) |
|----------|-----|--------------------------------|
| Режим TP ladder | `PAPER_TP_LADDER` | пусто (используется сетка) |
| Шаг сетки по PnL к средней | `PAPER_TP_GRID_STEP_PNL` | **0.05** |
| Доля продажи от **остатка** за ступень | `PAPER_TP_GRID_SELL_FRACTION` | **0.15** |
| Защита после 1-й ступени | `PAPER_TP_GRID_FIRST_RUNG_RETRACE_MIN_PNL` | **0.025** |
| Масштабы SL/TP по цене | `PAPER_TP_X` / `PAPER_SL_X` | **100** / **0** |
| Режим трейла | `PAPER_TRAIL_MODE` | **`ladder_retrace`** |
| Откат трейла | `PAPER_TRAIL_DROP` | **0.12** |
| Триггер «вооружения» трейла | `PAPER_TRAIL_TRIGGER_X` | **1.06** |
| Тайм-аут позиции | `PAPER_TIMEOUT_HOURS` | **4** |
| Шаг лога пика | `PAPER_PEAK_LOG_STEP_PCT` | **1** |

При **`PAPER_TP_REGIME_ENABLED=1`** к сетке применяются overrides из §7.2–7.3.

### 9.2 Live Oscar — режимы A и B

До фиксации отдельных блоков env в `ecosystem.config.cjs` для **`live-oscar`** использовать как **ориентир** текущие отличия live от paper (шаг сетки может совпадать; trail/timeout исторически мягче/дольше):

| Параметр | Референс (текущий live vs paper) |
|----------|-----------------------------------|
| `PAPER_TP_GRID_*` | как у paper (**step 5%**, **sell 15%**, **first rung retrace 2.5%**) до решения о разводе A/B |
| `PAPER_TRAIL_DROP` | **0.10** (live) vs **0.12** (paper) |
| `PAPER_TRAIL_TRIGGER_X` | **1.10** (live) vs **1.06** (paper) |
| `PAPER_TIMEOUT_HOURS` | **8** (live) vs **4** (paper) |

**Целевое состояние:** для **`live-oscar`** завести явные наборы параметров **«режим A»** и **«режим B»** (отдельные env-префиксы или дубли ключей с выбором в трекере после события усреднения) и описать их в [`W8.0_live_oscar_trading_bot.md`](./W8.0_live_oscar_trading_bot.md) при следующем bump.

## 10. Киты, создатель, агрессивные продавцы

Без изменений относительно предыдущей версии свода: общие для Oscar (paper/live) ключевые пороги:

- `PAPER_DIP_WHALE_ANALYSIS_ENABLED=1`, trigger не обязателен.
- Крупный продавец: **3000 USD**, capitulation **70%**, групповые продажи **5000 USD / 2 продавца / 40% dump**.
- Блок creator dump: вкл., lookback **20** мин, доля **5–60%**.
- Предиктор DCA / агрессор: пороги sells 24h, интервалы, chunk USD — как в `ecosystem.config.cjs`.
