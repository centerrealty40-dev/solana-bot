# Дорожная карта: контур данных для smart-money (Solana Alpha)

**Область:** аналитический пайплайн (ingest кошельков → нормализация → теги → жизненный цикл → сигналы).  
**Единая точка входа для агентов:** [`README.md`](./README.md).

---

## 0. Цель и границы

**Цель:** устойчивый **data plane** для будущей базы «smart money» на мемкоинах: надёжный ingest адресов (Gecko + RPC и др.), **версионируемые** следы происхождения, слой **тегов** с явной семантикой, **жизненный цикл** записей (включая неактивность >90 дней), затем правила кворума/сигналов.

**Вне первой волны:** финальная торговая логика live-бота, юридический комплаенс (учитывать при политике удаления данных).

---

## Фаза A — Контракты (до массового залива)

### A1. Глоссарий

- **Кошелёк** — pubkey Solana.  
- **Кластер сущности** — объединение кошельков одного оператора (позже).  
- **Seed** — адрес попал в систему из коллектора до полной верификации.  
- **Тег** — атрибут с типом, версией правила и периодом действия.

### A2. Логические слои данных

| Слой | Назначение |
|------|------------|
| **Staging / raw seed** | Сырьё коллектора с `collector_semver`, `batch_id`. |
| **Canonical `wallets`** | Дедуп + слияние источников. |
| **Tags** | Текущие и исторические классификации. |
| **Marts** | Агрегаты для скоринга и кворума (позже). |

### A3. Версионирование коллекторов

Обязательно для постфактум-анализа: **collector_id**, **semver (MAJOR.MINOR.PATCH)**, **batch_id**, **ingested_at** на каждой записи seed / в provenance. Семантика semver — см. [`CONCEPT_DATA_PLANE_STAGING_TAGS_LIFECYCLE.md`](./CONCEPT_DATA_PLANE_STAGING_TAGS_LIFECYCLE.md).

---

## Фаза B — Ingest (коллекторы)

- Локальный пилот → VPS с отдельным QuickNode-проектом.  
- Лимиты: на пул, на прогон, на сутки; один writer; очередь кандидатов.  
- Спека минимального Raydium-контура: [`W6.7_gecko_raydium_wallet_seed_collector_local.md`](./W6.7_gecko_raydium_wallet_seed_collector_local.md).

**Риски:** перегруз RPC/Postgres; общий API-ключ с торговыми ботами → изоляция endpoint и env-префиксов.

---

## Фаза C — Нормализация

- Единая строка на адрес в `wallets`; слияние seed + `swaps` без потери происхождения.  
- Рекомендуется таблица **provenance** для аудита поколений данных.

**Риски:** потеря истории при слепом UPDATE.

---

## Фаза D — Система тегов

- Классы тегов: structural/negative, behavioral, quality, outcome (позже).  
- Версии **правил** (`rule_id`, `rule_semver`); очередь задач на пересчёт; идемпотентность.  
- Разведение **current** vs **history** (или `revoked_at`).  
- Приоритеты при конфликте тегов — документировать явно.

**Риски:** дрейф семантики; дорогой full-scan → батчи и инкремент.

Детали: [`CONCEPT_DATA_PLANE_STAGING_TAGS_LIFECYCLE.md`](./CONCEPT_DATA_PLANE_STAGING_TAGS_LIFECYCLE.md).

---

## Фаза E — Жизненный цикл и неактивность

- Метрика «молчания» — по **вашим** таблицам наблюдений (`swaps`, профиль Atlas), не «нет tx в сети».  
- Эскалация: stale → archive → опционально soft-delete.  

**Риски:** удаление полезных связей; комплаенс → предпочитать soft-delete.

---

## Фаза F — Downstream (smart money / кворум)

- White/black множества после стабилизации тегов.  
- Кворум с учётом **кластера как одной голосующей единицы**.  
- Бэктест и мониторинг дрейфа скоров.

---

## Фаза G — Операции

- Runbooks: пауза ingest, replay батча, откат MAJOR-правил тегов.  
- Изменения нормативки — [`../strategy/release/RELEASE_OPERATING_MODEL.md`](../strategy/release/RELEASE_OPERATING_MODEL.md).

---

## Матрица концептуальных рисков

| Риск | Митигация |
|------|-----------|
| Смещение выборки (только Gecko new_pools) | Документировать покрытие; добавить источники |
| Несовместимость поколений данных | Collector semver + batch_id; версии правил тегов |
| Засор canonical | Staging → promote; лимиты seed |
| Ошибки кластеризации | Версии правил; консервативный merge |
| Путаница «молчание» vs «нет данных» | Учитывать только локальные факты |

## Порядок внедрения (чеклист)

1. **A** — глоссарий, модель таблиц, контракт версий.  
2. **B** — коллектор(ы) с лимитами.  
3. **C** — merge в canonical, provenance.  
4. **D** — очередь тегирования, current/history.  
5. **E** — политика 90d → stale → archive.  
6. **F** — кворум и сигналы.  
7. **G** — runbooks.
