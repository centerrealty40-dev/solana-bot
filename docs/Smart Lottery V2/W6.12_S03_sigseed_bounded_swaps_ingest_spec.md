# W6.12 — S03: Sigseed как узкий источник `swaps` (без stream)

**Тип:** normative implementation spec  
**Зависимости:** [`W6.12_S01_unified_rpc_budget_counter_spec.md`](./W6.12_S01_unified_rpc_budget_counter_spec.md)  
**Связанный обзор:** [`W6.12_OVERVIEW_detective_without_chain_firehose.md](./W6.12_OVERVIEW_detective_without_chain_firehose.md)  
**Исполнение на `v2`:** пошаговый backlog — [`W6.12_S03_sigseed_execution_spec.md`](./W6.12_S03_sigseed_execution_spec.md).

---

## 1. Цель

Зафиксировать **целевой** контур **pipeline 2** (`signatures_seed_queue` → worker → `rpc_features` → `sigseed:parse` → **`swaps`**) как **основной mint/signature-ориентированный** путь наполнения **`swaps`** без `sa-stream` (на текущей ветке `v2` код этого контура может отсутствовать — см. execution-spec).

Зафиксировать **операторские и программные лимиты**, чтобы enqueue не раздувал RPC при включении cron.

## 2. Текущее состояние (anchor)

Описание процессов и cron — [`deploy/RUNTIME.md`](../../deploy/RUNTIME.md): `sigseed:enqueue`, `sa-sigseed-worker`, `sigseed:parse`, лимиты Atlas → очередь (`ATLAS_ENQUEUE_*`).

Данная спека **не дублирует** RUNTIME построчно, а **дополняет** нормативными требованиями под цель W6.12.

## 3. Нормативные требования

### 3.1. Enqueue (что имеет право попадать в очередь)

1. Любой новый enqueue-путь должен иметь **явный дневной потолок** записей (уже есть `ATLAS_ENQUEUE_MAX_PER_DAY` — сохранить обязательность).
2. Запрещено включать «широкие» источники без второго лимита (например все минты из огромного списка без priority sampling).
3. Приоритеты:
   - минты, связанные с **новыми пулами** из торговых коллекторов;
   - минты из **ручного denylist / blacklist**;
   - минты, уже попавшие в **кандидаты** scam-farm (опционально, второй этап).

### 3.2. Worker + parse

1. Каждый billable RPC вызов worker/parse должен учитываться в **S01** (`component_id=sigseed_worker`).
2. При превышении глобального дневного cap worker завершает batch и откладывает остаток очереди (**без busy-spin retry**).
3. Логи очереди: сохранить/расширить метрики `queueDepth`, хвост `rpc_features` до парсера (как в RUNTIME).

### 3.3. Связь с детективом

- Детектив (`rug_cohort`, `orchestrated_split`) потребляет **`swaps`** по **rug-минтам** (`tokens.blacklisted` и эвристика ликвидности). Sigseed обязан **не ломать** семантику колонок `swaps` (dex, side, wallet, base_mint, quote_mint, время).

## 4. ENV — контракт документирования

Зафиксировать в `.env.example` и в этой спеке таблицу **обязательных** для прод режима переменных:

| Переменная | Назначение |
|------------|------------|
| `ATLAS_ENQUEUE_MAX_PER_DAY` | верхняя крышка постановок из Atlas/runner |
| `ATLAS_ENQUEUE_SWAPS_CEILING` | не ставить в очередь если уже достаточно свопов по минту |
| `SIGSEED_*` / существующие лимиты worker | не ослаблять без пересмотра S01 |

(Точные имена — сверить с кодом enqueue/worker при имплементации задачи.)

## 5. Критерии приёмки

1. Документ RUNTIME обновлён перекрёстной ссылкой на W6.12 S03 (одна строка достаточно).
2. В коде enqueue/worker подтверждено использование глобального ledger S01 **или** создана задача с явным ID (если S01 ещё не готов — временный флаг `SA_SIGSEED_DEFER_GLOBAL_LEDGER=1` запрещён на прод).
3. Операторский сценарий: при лимите 1.5M кредитов суммарно с оркестратором и S02 очередь sigseed **стабилизируется** (не растёт бесконечно из-за retry-storm).

## 6. Non-goals

- Замена sigseed полным сканом кошелька — это **S02**.
- Поддержка всех DEX в parse — только по мере существующего `parse-rpc-tx-swap`; расширение — отдельные спеки.
