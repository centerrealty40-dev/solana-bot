**Назначение:** пошаговый обязательный минимум для агента-реализатора, чтобы детектив был **рабочей v1**, а не только библиотекой.  
**Нормативная архитектура:** [`W6.9_wallet_intel_detective_trading_spec.md`](./W6.9_wallet_intel_detective_trading_spec.md).

## 1. Мелкие доработки для «рабочей v1» (обязательный чеклист агента)

Ниже — то, без чего реализация легко останется «библиотекой без входа» или сломается на первом прогоне. Агент должен закрыть **все** пункты блока **MVP**.

### 1.1 Точки входа (CLI / npm)

| Артефакт | Требование |
|----------|------------|
| **`package.json` scripts** | Явные команды: например `wallet-intel:policy`, `wallet-intel:mint-check`, `scam-farm:detect`, опционально `wallet-intel:pipeline` (обёртка в правильном порядке). Сейчас `runScamFarmDetectivePass` существует без обязательного npm-алиаса — **добавить**. |
| **`src/scripts/wallet-intel-*.ts`** (или один `wallet-intel.ts` с подкомандами) | `dotenv/config`, парсинг argv (`--dry-run`, `--rule-set`, `--mint`, `--limit`), корректный `process.exit` коды (0/1). |
| **`--help`** | Краткая справка по env и флагам (оператор не должен лезть в код). |

### 1.2 Миграция БД и Drizzle

| Артефакт | Требование |
|----------|------------|
| **`src/core/db/migrations/NNNN_wallet_intel*.sql`** | Таблицы из **§2** этого файла + индексы `(address)`, `(decision, rule_set_version)`, `(computed_at DESC)`. |
| **`schema.ts`** | Экспорт таблиц для Drizzle; типы для приложения. |
| **`npm run db:migrate`** | Прогон на чистой БД документировать одной строкой в комментарии к PR. |

### 1.3 Конфигурация и безопасность по умолчанию

| Артефакт | Требование |
|----------|------------|
| **`.env.example`** | Блок `WALLET_INTEL_*`, `SCAM_FARM_*`, ссылки на `DATABASE_URL`; дефолты: **`SCAM_FARM_DRY_RUN=1`**, **`SCAM_FARM_WRITE_ATLAS=0`** до ручного включения. |
| **`WALLET_INTEL_RULE_SET_VERSION`** | Строка semver или `v0.1.0`; любая материализация решений пишет её в строку. |
| **Лимиты** | `LIMIT` на батч адресов и на SQL в духе существующего `SCAM_FARM_MAX_SQL_ROWS`, чтобы первый прогон не положил Postgres. |

### 1.4 Связка с существующим кодом (ловушки)

| Тема | Что сделать |
|------|-------------|
| **`wallet-tagger`** | Функция **`tagAtlas`** читает только **`entity_wallets`** (`profile_updated_at`). Адреса из **`wallets`** (W6.8) там могут **отсутствовать**. Для v1: либо **батч «promote»** `wallets → entity_wallets` (минимальный upsert профиля), либо новый режим **`tagWalletsTableSince(batch)`**, либо документированная зависимость от **`atlas`/trace**, который заполняет `entity_wallets`. Без этого теги не покроют seed-базу. |
| **`scam-farm-detective`** | Уже пишет **`scam_farm_candidates`**; политика должна **читать** кандидатов + статусы + `wrote_to_atlas`, а не дублировать SQL правил в первой версии (см. §6.2 основной спеки). |
| **`INSERT wallets`** | Оркестратор использует **`ON CONFLICT DO NOTHING`** — политика не должна ожидать обновления `metadata` при повторном seed. |

### 1.5 Идемпотентность и аудит

| Артефакт | Требование |
|----------|------------|
| **`wallet_intel_runs`** (или поле в логе) | На каждый прогон: `started_at`, `finished_at`, `rule_set_version`, счётчики inserted/updated/skipped, `git_sha` опционально. |
| **Upsert решений** | Уникальный ключ `(wallet_address, rule_set_version)` или upsert с заменой **`computed_at`** только если score вырос (политика зафиксировать). |
| **JSONL / structured logs** | Одна строка JSON на завершение прогона (как у оркестратора) → простой мониторинг без Datadog. |

### 1.6 Тесты (минимум для merge)

| Тест | Что проверяет |
|------|----------------|
| **Unit** | Чистая функция `mintDecision(earlyBuyers, decisionsMap, ignoreSet)` по §7 основной спеки — без БД. |
| **Unit** | Маппинг тегов → `BLOCK_TRADE` / `IGNORE` (таблица из §12 основной спеки после фиксации). |
| **Интеграция (опционально)** | С docker-compose Postgres + фикстуры: один mint, два покупателя, один BLOCK → `NO_TRADE`. |

### 1.7 Наблюдаемость и эксплуатация

| Артефакт | Требование |
|----------|------------|
| **Preflight SQL** | Скрипт или подкоманда `wallet-intel:doctor`: счётчики `wallets`, `swaps` за 24h, `money_flows` за 24h, `scam_farm_candidates`; если `swaps_recent = 0` — **exit 2** с текстом «MINT_DECISION недоступен». |
| **PM2 / cron** | Заготовка в `ecosystem.config.cjs` или одна строка в `deploy/RUNTIME.md`: частота **не чаще**, чем успевает ingest (например 1×/сутки для policy + по требованию mint-check). |
| **Алерт** | При ошибке миграции или doctor — не молчать (хотя бы stderr + non-zero exit для cron). |

### 1.8 Документация для следующего агента

| Артефакт | Требование |
|----------|------------|
| **Этот файл + README V2** | «Как запустить v1 за 5 минут»: см. **§3** ниже. |
| **CHANGELOG / VERSION** | MINOR bump продукта при появлении таблиц и публичных CLI (как принято в репозитории). |

## 2. Минимальная схема таблиц (v1, черновик для миграции)

Названия можно уточнить; смысл столбцов — зафиксирован.

### 2.1 `wallet_intel_decisions`

| Столбец | Тип | Описание |
|---------|-----|----------|
| `wallet_address` | `varchar(64)` PK часть | |
| `rule_set_version` | `text` PK часть | |
| `decision` | `text` | `BLOCK_TRADE`, `SMART_TIER_A`, `IGNORE_FOR_SIGNAL`, `UNKNOWN`, … |
| `score` | `numeric` или `real` | 0–100 для сортировки/отладки |
| `reasons` | `jsonb` | массив строк причин |
| `sources` | `jsonb` | ссылки: `candidate_id`, `tag`, `cluster_id` |
| `computed_at` | `timestamptz` | |

Индекс: `(decision, rule_set_version)`, `(computed_at DESC)`.

### 2.2 `wallet_intel_runs`

| Столбец | Описание |
|---------|----------|
| `id` | bigserial PK |
| `rule_set_version` | text |
| `started_at` / `finished_at` | timestamptz |
| `metrics` | jsonb (counts, duration_ms) |
| `status` | `ok` / `failed` |
| `error` | text nullable |

### 2.3 Опционально v1.1

- **`mint_intel_cache`** `(mint, rule_set_version, decision, computed_at)` — если запросы к mint повторяются и дороги по JOIN.

## 3. Порядок первого end-to-end прогона (~5 минут после реализации)

1. `npm run db:migrate`
2. `npm run wallet-intel:doctor` — убедиться, что есть недавние **`swaps`** (или принять осознанный режим только БД-кандидатов без mint-gate).
3. Обеспечить покрытие тегов для seed: **promote `wallets` → `entity_wallets`** или режим теггера по `wallets` (**§1.4**).
4. `SCAM_FARM_DRY_RUN=1 npm run scam-farm:detect` — проверить SQL и метрики.
5. `npm run wallet-intel:policy -- --dry-run` — без записи или в транзакции rollback (если реализовано).
6. Снять dry-run, прогнать policy; выборочно `wallet-intel:mint-check --mint <pk>`.

## 4. Явные non-goals для первой рабочей версии

- Полный граф Louvain по всей БД.
- Автоматическая покупка/продажа (только данные для стратегии).
- Слияние дубликатов кластеров без человеческого runbook.
- Зависимость от Helius (Phase 3 по §8 основной спеки).
