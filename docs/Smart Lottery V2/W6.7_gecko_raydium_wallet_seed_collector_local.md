# W6.7 — Gecko `new_pools` → Raydium → QuickNode: локальный коллектор пополнения `wallets`

**Расположение:** `docs/Smart Lottery V2/` — см. [`README.md`](./README.md).  
**Статус:** normative spec (реализация по желанию; v1 — только локальная машина).  
**Контекст продукта:** `solana-alpha`. Репозиторий: `Ideas/solana-alpha`. Целевая ветка разработки продукта: **`v2`** (GitHub).  
**Зависимости:** W6.0 (схема БД с таблицей `wallets`), общие паттерны коллекторов (`scripts-tmp/*-collector.mjs`), `deploy/RUNTIME.md` (конвейеры RPC не смешивать без явного решения).

---

## 1. Цель

Минимально и воспроизводимо **наполнять** таблицу **`wallets`** адресами, связанными с **новыми пулами Raydium**, обнаруженными через **GeckoTerminal `new_pools`**, с использованием **QuickNode** (JSON-RPC).  

**Окружение v1:** только **локальный** компьютер оператора (отдельный `.env`, dev Postgres или Docker). Без PM2/cron на VPS до отдельного решения и записи в [`CHANGELOG`](../strategy/release/CHANGELOG.md).

---

## 2. Границы и запреты (Forbidden actions)

1. **Не вызывать DexScreener** из этого коллектора (не делить квоту/риск с торговыми ботами, завязанными на DS).
2. **Не менять** торговые коллекторы, cron sigseed и **`enqueue-seed-signatures`** в рамках этой задачи без отдельной spec.
3. **Не писать** в `entity_wallets`, `money_flows`, `swaps` в v1 — только **`wallets`** (+ опциональная локальная очередь).
4. **Не пушить в `main`**; изменения по реализации — через **`v2`** по [`RELEASE_OPERATING_MODEL.md`](../strategy/release/RELEASE_OPERATING_MODEL.md).
5. **Не параллелить** обработку пулов и не открывать «шторм» RPC: один процесс, последовательный цикл.
6. **Не использовать** общую платформенную папку `docs/platform/**` для этой feature — только продуктовые docs под `solana-alpha/docs/`.

## 3. Источники данных

### 3.1 GeckoTerminal (HTTP)

- **Endpoint:** `GET https://api.geckoterminal.com/api/v2/networks/solana/new_pools?page={n}`  
- **Пагинация:** параметр окружения **`SA_GRWS_GECKO_PAGES_MAX`** (малое значение по умолчанию для локали, напр. 1–2).
- **Интервал:** локально — режим **`--once`** или редкий таймер (напр. 5–15 мин); без перекрывающихся тиков (`skip if previous tick active`).

### 3.2 Фильтр Raydium

- Обрабатывать элементы `data[]` ответа Gecko.
- **Включать** пул, если в объединённом тексте полей (`dex_name`, `name`, `address`, и т.п., в нижнем регистре) есть признак **Raydium** (аналогично эвристике в `raydium-collector.mjs` / `orca-collector.mjs`).
- **Извлекать минимум:** `pool_address` (Solana pubkey пула), `base_mint`, `quote_mint`, время прохода коллектора.

---

## 4. Выбор адреса для `getSignaturesForAddress`

**Зафиксировано для v1:** использовать только **`pool_address`** из Gecko (не `base_mint`).

| Вариант | Обоснование |
|---------|-------------|
| **Пул** | Подписи отражают активность рынка; меньше постороннего шума, чем по mint. |
| Mint | Шире, но сильно шумнее и дороже по RPC; не использовать в v1. |

---

## 5. QuickNode (JSON-RPC)

- **URL:** `SOLANA_RPC_HTTP_URL` или изолированный **`SA_GRWS_RPC_URL`** (рекомендуется отдельный ключ/endpoint для экспериментов).
- **Обязательный метод:** `getSignaturesForAddress` по `pool_address`, с лимитом страниц **`SA_GRWS_SIG_PAGES_MAX`** и параметром `limit` согласно возможностям ноды.

### 5.1 Ограничение «один метод» vs заполнение `wallets`

Ответ `getSignaturesForAddress` **не содержит** надёжного списка pubkey пользователей без разбора транзакций.

Спека допускает два подрежима; переключение — **`SA_GRWS_MODE`** (см. §8).

| Подрежим | RPC | Запись в `wallets` |
|----------|-----|-------------------|
| **v1a** | только `getSignaturesForAddress` | Нет (только лог/очередь подписей для последующего этапа). |
| **v1b** | подписи + до **`SA_GRWS_MAX_TX_FETCHES_PER_POOL`** вызовов **`getTransaction`** (`encoding: jsonParsed`, `maxSupportedTransactionVersion: 0`) | Да — см. §6.6 (извлечение pubkey). |

**Зафиксировано для пилота:** значение по умолчанию **`SA_GRWS_MODE=v1b`**. Режим **`v1a`** только для отладки RPC без записи в БД.

## 6. Запись в БД

### 6.1 Целевая таблица

- **`wallets`** (см. `src/core/db/schema.ts`): PK `address`, `first_seen_at`, `metadata` jsonb.

### 6.2 Идемпотентность

- Пилот (**§6.6 PI-9**): только **`ON CONFLICT (address) DO NOTHING`**.  
- Будущее: аккуратный `DO UPDATE` для merge `metadata` — отдельная версия коллектора и правка спеки.

### 6.3 Контракт `metadata` (полный контракт для пилота)

Каждая вставленная строка **`wallets`** (режим **v1b**) содержит в **`metadata`** минимум следующие ключи:

```json
{
  "gecko_raydium_seed": true,
  "seed_pool": "<pool_address>",
  "seed_base_mint": "<mint>",
  "seed_quote_mint": "<mint_or_null>",
  "seed_ts": "<ISO-8601>",
  "collector_id": "sa-grws",
  "collector_semver": "0.1.0",
  "batch_id": "20260430T153045Z_a3f9e21c",
  "sa_grws_mode": "v1b"
}
```

Семантика **`collector_id` / `collector_semver` / `batch_id`**: см. [`CONCEPT_DATA_PLANE_STAGING_TAGS_LIFECYCLE.md`](./CONCEPT_DATA_PLANE_STAGING_TAGS_LIFECYCLE.md).  
**Пилот:** отдельная staging-таблица **не** используется — только **`wallets`** и этот JSON; закрытые правила — §6.6.

### 6.4 Опциональная очередь (локально)

Таблица в духе существующих очередей продукта (`CREATE TABLE IF NOT EXISTS` в скрипте):  

`pool_address` PK, `status`, `attempts`, `last_error`, `added_at`, `processed_at`.

### 6.5 Нагрузка на Postgres

- Батч-вставки (сотни–тысячи строк на транзакцию).
- Один писатель, без параллельных воркеров v1.

**Схема БД:** только текущая БД продукта (`solana_alpha`); отдельную схему `smart_money` из `DB_TOPOLOGY` платформы Ideas **не вводить**, пока фича не промотируется в отдельный продукт.

### 6.6 Pilot implementation defaults (normative — микро-решения закрыты)

| ID | Решение |
|----|---------|
| **PI-1** | **Режим RPC:** по умолчанию **`v1b`**. Переменная **`SA_GRWS_MODE`**: `v1b` \| `v1a`. |
| **PI-2** | **Куда писать:** только таблица **`wallets`**, без отдельной staging-таблицы в **v0.1.0** пилота. Очередь пулов в Postgres — опционально позже. |
| **PI-3** | **`collector_id`:** константа в коде **`sa-grws`**, переопределение через **`SA_GRWS_COLLECTOR_ID`**. |
| **PI-4** | **`collector_semver`:** env **`SA_GRWS_COLLECTOR_SEMVER`**, значение по умолчанию **`0.1.0`**; bump вручную по правилам semver из концепта при смене семантики извлечения/фильтра. |
| **PI-5** | **`batch_id`:** генерируется **один раз на запуск процесса** (один прогон `--once` или один старт цикла): строка **`${ISO8601_UTC_compact}Z_${8_hex_lowercase}`**, например `20260430T153045Z_a3f9e21c`. |
| **PI-6** | **Извлечение адресов из ответа `getTransaction`:** множество pubkey = все записи **`transaction.message.accountKeys`** (или эквивалент в ответе ноды), у которых **`signer === true`**. Если структура не JSON-parsed ожидаемым образом — fallback: **`meta.feePayer`** (или поле fee payer из legacy layout в ответе QuickNode). Дубликаты в рамках одной tx убрать. |
| **PI-7** | **Исключения:** не добавлять в результат pubkey, если он входит в **дефолтный статический список program id** в коде (минимум: основные program Raydium v4/CPMM — конкретные base58 зашить в константу **`SA_GRWS_BUILTIN_IGNORE_PROGRAMS`** в модуле коллектора; расширение через **`SA_GRWS_IGNORE_PROGRAM_IDS`** — comma-separated список дополнительных pubkey). Программы не путаем с пользовательскими кошельками. |
| **PI-8** | **Валидация:** принимать только строки длиной **32–44** символов в base58-алфавите Solana (как эвристика «похоже на pubkey»). |
| **PI-9** | **`ON CONFLICT (address)`:** в пилоте **`DO NOTHING`** (не мержить `metadata` при существующей строке). Расширение с merge — отдельная minor-версия коллектора и правка спеки. |
| **PI-10** | **Артефакт:** один файл **`scripts-tmp/sa-grws-collector.mjs`** (или эквивалентное имя с префиксом `sa-grws`), стиль как у `raydium-collector.mjs` (`pg` Pool, JSON-логи). Команда запуска: **`npm run sa-grws-collector`** (добавить в `package.json` при реализации). |

## 7. Изоляция и конкуренция

- Отдельный префикс env **`SA_GRWS_*`** — не пересекать с торговым конфигом.
- Лимиты Gecko/RPC настраиваются консервативно; опционально файл-счётчик **`SA_GRWS_RPC_DAILY_SOFT_CAP`** для защиты квоты QuickNode при отладке.

---

## 8. Переменные окружения (контракт пилота)

| Переменная | Назначение | Default (пилот) |
|------------|------------|-----------------|
| `DATABASE_URL` | Локальный Postgres. | — |
| `SA_GRWS_RPC_URL` | QuickNode (предпочтительно отдельный endpoint). | если пусто — fallback **`SOLANA_RPC_HTTP_URL`** |
| `SA_GRWS_MODE` | `v1b` \| `v1a`. | **`v1b`** |
| `SA_GRWS_COLLECTOR_ID` | Идентификатор в `metadata`. | **`sa-grws`** |
| `SA_GRWS_COLLECTOR_SEMVER` | Версия коллектора в `metadata`. | **`0.1.0`** |
| `SA_GRWS_IGNORE_PROGRAM_IDS` | Доп. program id через запятую для PI-7. | пусто |
| `SA_GRWS_GECKO_PAGES_MAX` | Макс. страниц `new_pools` за прогон. | **`2`** |
| `SA_GRWS_SIG_PAGES_MAX` | Макс. страниц подписей на пул. | задать консервативно, напр. **`5`** |
| `SA_GRWS_MAX_POOLS_PER_RUN` | Макс. пулов Raydium за прогон. | напр. **`10`** |
| `SA_GRWS_MAX_TX_FETCHES_PER_POOL` | Макс. `getTransaction` на пул (только `v1b`). | напр. **`30`** |
| `SA_GRWS_RPC_SLEEP_MS` | Пауза между RPC-вызовами. | напр. **`250`** |
| `SA_GRWS_DRY_RUN` | `1` — без записи в БД. | `0` |

## 9. Наблюдаемость

Структурированные логи (как у `*-collector.mjs`): `poolsSeen`, `poolsRaydium`, `signaturesPages`, `txFetched`, `walletsInserted`, `errors`, длительность тика.

## 10. Acceptance criteria (локально)

1. При наличии данных Gecko в логе **ненулевой** `poolsRaydium` (или явная причина нуля — фильтр/пустая выдача).
2. При `SA_GRWS_DRY_RUN=0` и **`SA_GRWS_MODE=v1b`** — новые строки в **`wallets`** с `metadata` по контракту §6.3 (включая `collector_*`, `batch_id`).
3. Повторный прогон не создаёт дубликатов PK `wallets`.
4. Нет параллельной обработки множества пулов в одном процессе v1.
